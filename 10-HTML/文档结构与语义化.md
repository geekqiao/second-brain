# 文档结构与语义化

> [!info] 所属：[[HTML MOC]] · 下一篇 [[文本链接与列表]]

## 核心概念

一个 HTML 文档由固定骨架构成。**语义化（semantic）** 指的是用表达「含义」的标签（如 `article`、`nav`）代替万能的 `div`，让浏览器、搜索引擎、辅助技术都能读懂页面结构。

### 标准文档骨架
- `<!DOCTYPE html>`：声明为 HTML5 文档。
- `<html lang="zh-CN">`：根元素，`lang` 帮助搜索引擎与读屏器正确发音。
- `<head>`：元信息（不显示），如字符集、viewport、标题、CSS/JS 引用。
- `<body>`：页面可见内容。

### 语义化结构标签
| 标签 | 语义 |
| --- | --- |
| `<header>` | 页头 / 区块头部（常含 logo、导航） |
| `<nav>` | 主要导航链接 |
| `<main>` | 页面主体，每页唯一 |
| `<article>` | 可独立分发/复用的内容（一篇文章、一个帖子） |
| `<section>` | 主题相关的内容区块，通常带标题 |
| `<aside>` | 旁支内容（侧边栏、广告、相关推荐） |
| `<footer>` | 页脚（版权、联系、链接） |

### 为什么语义化
1. **SEO**：搜索引擎更准确理解内容权重。
2. **可访问性**：读屏器可按地标（landmark）跳转。
3. **可维护性**：`<header>` 比 `<div class="header">` 更直观。

## 代码示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">                 <!-- 字符编码 -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- 移动端适配 -->
  <title>我的博客</title>
</head>
<body>
  <header>
    <h1>我的博客</h1>
    <nav>
      <a href="/">首页</a>
      <a href="/about">关于</a>
    </nav>
  </header>

  <main>
    <article>
      <h2>文章标题</h2>
      <p>正文段落……</p>
      <section>
        <h3>小节标题</h3>
        <p>小节内容……</p>
      </section>
    </article>

    <aside>
      <h3>相关推荐</h3>
      <ul><li>另一篇文章</li></ul>
    </aside>
  </main>

  <footer>
    <p>© 2026 我的博客</p>
  </footer>
</body>
</html>
```

## ⚠️ 易错点 / 面试要点

- **`<section>` vs `<article>`**：`article` 独立完整（拿出去仍有意义），`section` 是主题分块。判断法：能否被 RSS 单独订阅？能→`article`。
- **一个页面只有一个 `<main>`、通常一个 `<h1>`**，否则稀释 SEO 权重。
- **`<div>` 不是禁用**：当没有合适语义标签时（纯布局容器）用 `div` 没错，但别用 `div` 包裹本该是 `nav`/`header` 的内容。
- **忘记 `<!DOCTYPE html>`**：浏览器会进入「怪异模式（quirks mode）」，渲染行为异常。
- **`lang` 属性**：影响翻译插件、读屏发音，别省。
