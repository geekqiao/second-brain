# 多媒体与嵌入

> [!info] 所属：[[HTML MOC]] · 上一篇 [[表单与输入]] · 下一篇 [[HTML5 新特性]]

## 核心概念

现代网页大量依赖图片、音视频。用好它们的关键是**性能**（别让大图拖慢页面）与**可访问性**（给替代文本）。

### `<img>` 关键属性
- `src`：图片地址。
- `alt`：**替代文本**，图片加载失败或被读屏器朗读时使用。装饰性图片用空 `alt=""`。
- `loading="lazy"`：懒加载，进入视口才加载，节省流量。
- `width` / `height`：建议设置，**避免布局偏移（CLS）**。
- `decoding="async"`：异步解码，不阻塞主线程。

### 响应式图片
- `srcset` + `sizes`：根据屏幕宽度/像素密度选最合适的图，**避免在小屏上下载 4K 大图**。
- `<picture>` + `<source>`：支持格式回退（如 AVIF → WebP → JPG）和艺术指导（art direction，不同屏幕裁不同图）。

### 音视频
- `<video>` / `<audio>`：`controls`（控制条）、`autoplay`（慎用）、`loop`、`muted`、`preload`。
- `<source>`：提供多种格式，浏览器选第一个支持的。
- `<track>`：字幕（WebVTT），无障碍与国际化必备。

### 嵌入
- `<iframe>`：嵌入外部页面（地图、视频、第三方小部件）。注意安全（`sandbox`）与性能。
- `<figure>` + `<figcaption>`：图文组合，配图说明。

## 代码示例

```html
<!-- 基本图片：alt + 懒加载 + 尺寸 -->
<img src="photo.jpg"
     alt="一只橘猫趴在键盘上"
     width="800" height="600"
     loading="lazy"
     decoding="async">

<!-- 响应式图片：srcset 按视口选图 -->
<img src="small.jpg"
     srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
     sizes="(max-width: 600px) 480px, 800px"
     alt="风景照">

<!-- picture：格式回退 + 艺术指导 -->
<picture>
  <source srcset="hero.avif" type="image/avif">   <!-- 最优格式优先 -->
  <source srcset="hero.webp" type="image/webp">
  <source media="(max-width: 600px)" srcset="hero-mobile.jpg">
  <img src="hero.jpg" alt="首页大图">              <!-- 最终回退 -->
</picture>

<!-- 视频：多格式 + 字幕 -->
<video controls width="640" poster="cover.jpg">
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  <track kind="subtitles" src="subs-zh.vtt" srclang="zh" label="中文字幕">
  您的浏览器不支持 video 标签。
</video>

<!-- 图文组合 -->
<figure>
  <img src="chart.png" alt="2026 年度销售柱状图">
  <figcaption>图 1：各季度销售额对比</figcaption>
</figure>

<!-- iframe：限制权限更安全 -->
<iframe src="https://maps.example.com"
        sandbox="allow-scripts allow-same-origin"
        loading="lazy"></iframe>
```

## ⚠️ 易错点 / 面试要点

- **`alt` 不能省**：SEO 与可访问性都依赖它。纯装饰图用 `alt=""`（空），绝不要写 `alt="图片"` 这种废话。
- **大图不优化 = 慢**：始终提供压缩 + 现代格式（WebP/AVIF）+ `srcset`，移动端流量与首屏时间立竿见影。
- **设宽高防 CLS**：不写 `width/height`，图片加载完会撑开布局，导致「内容跳」。
- **`autoplay` 要配 `muted`**：浏览器禁止有声音的自动播放，用户体验也差。
- **iframe 是安全风险源**：第三方可被注入恶意脚本。用 `sandbox` 限制能力，必要时加 `allow` 白名单。
- **CORS**：从 canvas 读取跨域图片像素会被同源策略阻挡，需配 `crossorigin`。
