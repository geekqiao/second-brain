# 表单与输入

> [!info] 所属：[[HTML MOC]] · 上一篇 [[文本链接与列表]] · 下一篇 [[多媒体与嵌入]]

## 核心概念

表单是网页**收集用户输入**的唯一标准方式。HTML5 让 `input` 拥有丰富的类型与**原生校验**能力。

### `<form>` 关键属性
- `action`：提交目标 URL。
- `method`：`GET`（拼到 URL，适合查询）或 `POST`（放请求体，适合提交敏感/大量数据）。

### `<input>` 类型（type）
| type | 说明 | 移动端键盘优化 |
| --- | --- | --- |
| `text` / `password` | 文本 / 密码（掩码） | ✓ |
| `email` / `tel` / `url` | 带格式校验 | 弹出对应键盘 |
| `number` / `range` | 数字 / 滑块 | 数字键盘 |
| `date` / `time` / `month` | 日期时间选择器 | — |
| `checkbox` / `radio` | 复选 / 单选 | — |
| `file` | 文件上传 | — |
| `color` | 取色器 | — |
| `search` | 搜索框 | — |
| `hidden` | 不可见，存提交数据 | — |

### 配套元素
- `<label for="id">`：点击 label 等于聚焦输入框，**必配**（可访问性）。
- `<select>` + `<option>`：下拉选择；`<optgroup>` 分组。
- `<textarea>`：多行文本。
- `<fieldset>` + `<legend>`：把相关字段分组并命名。

### 原生校验属性
`required`（必填）、`pattern`（正则）、`minlength`/`maxlength`、`min`/`max`、`type=email` 自动校验格式。

## 代码示例

```html
<form action="/register" method="POST" novalidate>
  <!-- novalidate 可关闭原生校验以演示；实际保留即可 -->

  <fieldset>
    <legend>注册账号</legend>

    <p>
      <label for="email">邮箱：</label>
      <input type="email" id="email" name="email" required
             placeholder="you@example.com">
    </p>

    <p>
      <label for="pwd">密码：</label>
      <input type="password" id="pwd" name="pwd"
             minlength="8" required>
    </p>

    <p>
      <label for="age">年龄：</label>
      <input type="number" id="age" name="age" min="0" max="150">
    </p>

    <p>
      <label for="phone">手机号：</label>
      <input type="tel" id="phone" name="phone"
             pattern="1[3-9]\d{9}" title="请输入 11 位手机号">
    </p>

    <p>
      <label>性别：</label>
      <label><input type="radio" name="gender" value="m" checked> 男</label>
      <label><input type="radio" name="gender" value="f"> 女</label>
    </p>

    <p>
      <label for="city">城市：</label>
      <select id="city" name="city">
        <optgroup label="一线城市">
          <option>北京</option>
          <option>上海</option>
        </optgroup>
        <option>其它</option>
      </select>
    </p>

    <p>
      <label for="bio">简介：</label>
      <textarea id="bio" name="bio" maxlength="200"></textarea>
    </p>

    <p>
      <label><input type="checkbox" name="agree" required> 同意条款</label>
    </p>

    <button type="submit">提交</button>
    <button type="reset">重置</button>
  </fieldset>
</form>
```

## ⚠️ 易错点 / 面试要点

- **永远给 `<input>` 配 `<label>`**：用 `for` + `id` 绑定，或把 input 包在 label 内。无 label 的输入框读屏器无法朗读。
- **`name` 属性决定提交字段名**：没有 `name` 的字段不会被表单提交。
- **GET vs POST**：密码、文件、长内容必须用 POST；GET 会暴露在 URL 并被收藏/记录。
- **radio 同组必须同名**：一组单选框共享同一个 `name` 才能互斥。
- **原生校验只是第一道防线**：浏览器可绕过，服务端**必须再校验一次**（安全红线）。
- **文件上传别忘了 `enctype="multipart/form-data"`**：否则文件传不上去。
