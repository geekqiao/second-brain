# 可访问性 a11y

> [!info] 所属：[[HTML MOC]] · 上一篇 [[元数据与 SEO]]

## 核心概念

**a11y**（accessibility，11 个字母故缩写）指让**所有人**——包括视障、听障、运动障碍、认知障碍用户——都能使用你的网站。它不只是「为少数人」，也提升整体可用性（键盘操作、清晰对比、移动端体验），并在很多地区是**法律要求**。

可访问性有四大支柱（POUR）：可感知（Perceivable）、可操作（Operable）、可理解（Understandable）、健壮（Robust）。

### 语义化是无障碍的基础
正确的 HTML 标签（`<button>`、`<nav>`、`<main>`）自带语义，读屏器能识别。用 `<div onclick>` 模拟按钮，读屏器只会念「文本」，无法操作。详见 [[文档结构与语义化]]。

### ARIA（当原生 HTML 不够用时）
`role`（角色）、`aria-label`（可读标签）、`aria-hidden`（对辅助技术隐藏）、`aria-expanded`/`aria-disabled`（状态）。
> **第一法则：能用原生 HTML 就别用 ARIA。** ARIA 是补丁，不是替代。

### 替代文本
- 图片用 `alt` 描述含义。
- 装饰图用空 `alt=""` 隐藏。

### 键盘可达性
- 所有交互元素必须能用 `Tab` 到达、`Enter`/`Space` 激活。
- 可见焦点（focus ring）不要用 CSS 删掉。

### 颜色与对比度
- 文字与背景对比度建议 ≥ 4.5:1（WCAG AA）。
- 不要只用颜色传达信息（色盲用户看不到红/绿区分），加图标或文字。

## 代码示例

```html
<!-- ❌ 不好：div 当按钮，键盘不可达，读屏不识别 -->
<div onclick="save()" class="btn">保存</div>

<!-- ✅ 好：原生 button -->
<button type="button" onclick="save()">保存</button>

<!-- 图标按钮：读屏器念不出图标，给 aria-label -->
<button aria-label="关闭" onclick="close()">
  <svg aria-hidden="true">…✕…</svg>
</button>

<!-- 跳过导航（skip link）：键盘用户的快捷入口 -->
<a href="#main" class="skip-link">跳到主内容</a>
<nav>…大量导航…</nav>
<main id="main">…正文…</main>

<!-- ARIA 动态区域：内容更新时通知读屏器 -->
<div role="status" aria-live="polite">
  <!-- 如：表单校验错误、新消息提示 -->
</div>

<!-- 表单错误要关联（aria-describedby） -->
<label for="email">邮箱</label>
<input id="email" type="email" aria-describedby="email-err" aria-invalid="true">
<span id="email-err" role="alert">请输入有效邮箱</span>
```

```css
/* 别删 focus 轮廓；想要自定义就显式提供 */
button:focus-visible {
  outline: 3px solid #2563eb;
  outline-offset: 2px;
}
```

## ⚠️ 易错点 / 面试要点

- **ARIA 第一法则**：能用原生语义标签就别加 ARIA。`<button>` 不需要 `role="button"`。
- **`alt` 三态**：有意义→描述；纯装饰→`alt=""`（隐藏）；可点击的图片链接→描述目标。
- **别删 `:focus` 样式**：键盘用户靠它定位。要美化就改 `:focus-visible`，别 `outline: none` 一刀切。
- **只用颜色 = 不可访问**：表单错误只标红不够，加文字或图标。
- **对比度**：浅灰字配白底是常见翻车点，用对比度检查器验证。
- **标题层级**：跳级（h1→h4）打乱读屏器的标题导航。
- **测试**：拔掉鼠标，只用键盘走一遍流程；用 Lighthouse / axe DevTools 扫描。
