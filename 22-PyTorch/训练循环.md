# 训练循环

> [!info] 所属：[[PyTorch MOC]] · 上一篇 [[构建模型 nn.Module]] · 下一篇 [[数据加载 Dataset & DataLoader]]

## 核心概念

PyTorch 不像 Keras 那样 `model.fit()` 一行搞定，它要求你**亲手写训练循环**——这看似繁琐，却给了完全的灵活性，也是理解「训练到底在干什么」的最佳途径。核心就五步，背下来一辈子受用。

### 训练一步的标准五步（每个 batch）
```python
optimizer.zero_grad()      # 1. 清零梯度（梯度是累加的）
logits = model(x)          # 2. 前向传播
loss = criterion(logits, y)# 3. 算损失
loss.backward()            # 4. 反向传播，算梯度
optimizer.step()           # 5. 用梯度更新参数
```

### 损失函数（criterion / loss）
衡量预测与真实的差距，训练就是最小化它。
| 任务 | 损失 |
| --- | --- |
| 回归 | `nn.MSELoss()`（均方误差） |
| 二分类 | `nn.BCEWithLogitsLoss()`（含 sigmoid） |
| 多分类 | `nn.CrossEntropyLoss()`（**内含 softmax**，故模型最后不加 softmax） |

### 优化器（optimizer）
用梯度更新参数的策略。
- **`SGD`**：基础梯度下降，配 `momentum` 更稳。
- **`Adam` / `AdamW`**：自适应学习率，**最常用**，默认首选。
- `lr`（学习率）：最重要的超参数。太大震荡发散，太小学不动。

### Epoch / Batch / Iteration
- **epoch**：把全部训练数据过一遍。
- **batch**：一次喂给模型的一小块数据。
- 一个 epoch = 若干 batch（iteration）。

### 评估与推理
- 评估前 `model.eval()` + `with torch.no_grad():`，关梯度、切行为。
- 不在训练集上评估（会过拟合虚高），用验证/测试集。

### 保存与加载
- **推荐存 `state_dict`**（参数字典）：`torch.save(model.state_dict(), path)`。
- 加载：建相同结构模型 → `model.load_state_dict(torch.load(path))`。
- 存整个模型对象（`torch.save(model, ...)`）依赖代码路径，不推荐。

## 代码示例

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

device = 'cuda' if torch.cuda.is_available() else 'cpu'

# 数据 + 模型（合成分类任务）
X = torch.randn(1000, 20); y = torch.randint(0, 3, (1000,))
loader = DataLoader(TensorDataset(X, y), batch_size=32, shuffle=True)

model = nn.Sequential(
    nn.Linear(20, 64), nn.ReLU(),
    nn.Linear(64, 3),
).to(device)

criterion = nn.CrossEntropyLoss()                          # 多分类（内含softmax）
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)  # Adam，lr=1e-3

# ============ 训练循环 ============
for epoch in range(10):
    model.train()                       # 训练模式
    total_loss = 0
    for xb, yb in loader:               # 遍历每个 batch
        xb, yb = xb.to(device), yb.to(device)

        optimizer.zero_grad()           # 1. 清零梯度
        logits = model(xb)              # 2. 前向
        loss = criterion(logits, yb)    # 3. 算损失
        loss.backward()                 # 4. 反向
        optimizer.step()                # 5. 更新参数
        total_loss += loss.item() * xb.size(0)
    print(f"epoch {epoch}: train loss = {total_loss/len(loader.dataset):.4f}")

    # ============ 评估 ============
    model.eval()                        # 评估模式（Dropout/BN 行为改变）
    correct = 0
    with torch.no_grad():               # 关梯度，省内存加速
        xb, yb = X.to(device), y.to(device)
        preds = model(xb).argmax(dim=1)
        correct = (preds == yb).sum().item()
    print(f"  acc = {correct/len(y):.4f}")

# ============ 保存 / 加载 ============
torch.save(model.state_dict(), 'model.pt')           # 存参数
# 加载
model2 = nn.Sequential(nn.Linear(20,64), nn.ReLU(), nn.Linear(64,3)).to(device)
model2.load_state_dict(torch.load('model.pt'))
model2.eval()
```

## ⚠️ 易错点 / 面试要点

- **五步顺序不能乱**：必须先 `zero_grad` 再 `backward` 再 `step`。漏 `zero_grad` → 梯度累加；漏 `step` → 参数不更新（loss 不降）。
- **`CrossEntropyLoss` 内含 softmax**：模型最后一层**不要**再接 `softmax`，否则等于算了两次。二分类用 `BCEWithLogitsLoss` 同理别先 sigmoid。
- **`loss.item()` 取标量**：记录 loss 要用 `.item()` 转成 Python float，否则会保留计算图、泄露内存。
- **评估忘 `eval()` + `no_grad()`**：Dropout 不关、还建梯度图，结果不准且浪费。
- **学习率是最关键超参**：loss 不降→lr 太小或太大；loss 爆 NaN→lr 太大。可用学习率调度器 `lr_scheduler`。
- **数据要 shuffle**：训练时 `DataLoader(shuffle=True)` 打乱，否则模型记住顺序、泛化差。
- **`model.train()` 切回**：评估后回到训练要切回 `train()`，否则 Dropout/BN 仍在评估态。
