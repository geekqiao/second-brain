# 张量基础

> [!info] 所属：[[PyTorch MOC]] · 下一篇 [[自动求导 autograd]]

## 核心概念

**张量（Tensor）** 是 PyTorch 的基本数据结构——多维数组，类似 NumPy 的 ndarray，但额外支持 **GPU 加速**和**自动求导**。所有神经网络的数据（输入、权重、梯度）都是张量。

### 创建张量
- 从数据：`torch.tensor([1, 2, 3])`（推断类型）。
- 从形状：`torch.zeros(...)`、`torch.ones(...)`、`torch.rand(...)`（随机）、`torch.randn(...)`（标准正态）。
- 从 NumPy：`torch.from_numpy(np_array)`（共享内存）。

### 形状与类型
- **dtype**：神经网络默认 `float32`（`torch.float32`），NumPy 默认 `float64`——混用易报类型不匹配。
- **device**：张量在 `cpu` 或 `cuda`（GPU）上。**参与运算的张量必须在同一设备**，跨设备会报错。
- **形状操作**：`reshape`/`view`（改变形状）、`squeeze`/`unsqueeze`（增删大小为 1 的维）、`permute`/`transpose`（交换维度）、`cat`/`stack`（拼接）。

### 运算
- 逐元素运算（`+ - * /`）、矩阵乘（`@` 或 `torch.matmul`）、聚合（`.sum()`/`.mean()`）。
- **广播（broadcasting）**：形状不同但兼容时自动扩展，规则同 NumPy。

### CPU ↔ GPU
- `.to('cuda')` / `.cuda()`：迁移到 GPU；`.cpu()`：回到 CPU。
- GPU 可用时才用：`device = 'cuda' if torch.cuda.is_available() else 'cpu'`。

## 代码示例

```python
import torch
import numpy as np

# 1) 创建
a = torch.tensor([1.0, 2.0, 3.0])          # float32
b = torch.zeros(2, 3)                       # 2x3 全零
c = torch.randn(3, 4)                       # 3x4 标准正态随机
arr = np.array([1, 2, 3])
d = torch.from_numpy(arr)                   # 从 numpy（共享内存）

# 2) 形状操作
x = torch.randn(2, 3, 4)
print(x.shape)                              # torch.Size([2, 3, 4])
flat = x.reshape(2, 12)                      # 展平：2x12
print(flat.shape)
y = x.view(6, 4)                             # view 要求内存连续，否则用 reshape
single = torch.randn(1, 5)
print(single.squeeze().shape)                # 去掉大小1的维 → [5]
print(single.unsqueeze(0).shape)             # 增一维 → [1,1,5]

# 3) 运算 + 广播
m = torch.rand(2, 3)
n = torch.rand(3)
print((m + n).shape)                         # [2,3]，n 广播成 [2,3]
print((m @ torch.rand(3, 2)).shape)          # [2,2] 矩阵乘
print(m.sum(dim=1))                          # 按行求和 → [2]

# 4) dtype（神经网络默认 float32）
x = torch.tensor([1, 2, 3], dtype=torch.float32)
print(x.dtype)

# 5) 设备迁移
device = 'cuda' if torch.cuda.is_available() else 'cpu'
x = x.to(device)
w = torch.randn(3, 3, device=device)         # 直接在目标设备创建
# y = x + w_on_cpu   # ❌ 报错：device 不一致
```

## ⚠️ 易错点 / 面试要点

- **设备不一致必报错**：参与运算的张量必须同 device（都在 CPU 或都在同一 GPU）。常见错误是模型在 GPU、数据还在 CPU。养成「数据和模型都 `.to(device)`」的习惯。
- **dtype 不匹配**：模型权重是 float32，输入是 float64/int 会报错。输入数据显式转 `.float()`。
- **`view` vs `reshape`**：`view` 要求张量内存连续（contiguous），不连续会报错；`reshape` 任何情况都能用（必要时拷贝）。不确定就用 `reshape`。
- **from_numpy 共享内存**：`torch.from_numpy(arr)` 与原 NumPy 数组**共享内存**，改一个另一个变。要独立副本用 `.clone()` 或 `torch.tensor(arr)`。
- **`dim`/`axis` 参数**：聚合时 `dim=0` 沿第 0 维（行方向，消去该维）。NumPy 叫 `axis`，PyTorch 叫 `dim`，含义相同。
- **GPU 张量转 NumPy**：必须先 `.cpu()` 再 `.numpy()`，GPU 张量不能直接 `.numpy()`。
- **整数除法**：PyTorch 整数张量 `/` 在不同版本行为不同，浮点运算前确保类型是 float。
