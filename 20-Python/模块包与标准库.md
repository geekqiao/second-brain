# 模块、包与标准库

> [!info] 所属：[[Python MOC]] · 上一篇 [[异步与并发]]

## 核心概念

Python 的**模块（module）** 是一个 `.py` 文件，**包（package）** 是含 `__init__.py` 的目录。`import` 系统决定了代码如何被组织和复用。**标准库（Standard Library）** 是 Python 自带的「电池」（batteries included），覆盖大量常用需求。**虚拟环境（venv）** 隔离项目依赖，避免全局污染。

### import 机制
- `import module` / `from module import name` / `import module as m`。
- 查找顺序：**当前目录 → 标准库 → 第三方（site-packages）→ PYTHONPATH**。
- `from package import item`：item 可以是子模块、子包，或 `__init__.py` 里定义的名字。
- 相对导入（包内部）：`from . import sibling` / `from .. import parent`。

### 包结构
```
myproj/
├── mypkg/
│   ├── __init__.py        # 包初始化（可空；Python 3.3+ 命名空间包可不写）
│   ├── mod_a.py
│   └── subpkg/
│       └── __init__.py
└── main.py
```

### `if __name__ == '__main__'`
- 被直接运行时 `__name__` 等于 `'__main__'`；被 import 时是模块名。
- 作用：让模块**既能被导入复用，又能独立运行**（常放测试/入口代码）。

### 虚拟环境（venv）
- 每个项目独立的 Python 环境，依赖互不干扰。
- `python -m venv .venv` 创建 → 激活 → `pip install`。

### 包管理工具
- **pip**：标准安装器。`requirements.txt` 列依赖。
- **现代替代**：`uv`（极快、Rust 写）、`poetry` / `hatch`（依赖+打包+发布一站式）。
- 锁文件：`requirements.txt`（pip）、`uv.lock`/`poetry.lock`（精确版本）。

### 必知标准库（精选）
| 库 | 用途 |
| --- | --- |
| `os` / `pathlib` | 系统/路径操作（`pathlib` 更现代） |
| `sys` | 解释器信息、命令行参数 |
| `json` | JSON 读写 |
| `re` | 正则表达式 |
| `datetime` | 日期时间 |
| `collections` | `Counter`/`defaultdict`/`namedtuple` |
| `itertools` | 迭代器工具（组合/笛卡尔积） |
| `functools` | `lru_cache`/`partial`/`wraps` |
| `subprocess` | 调外部命令 |
| `logging` | 日志 |
| `unittest` / `pytest` | 测试 |

## 代码示例

```python
# mypkg/math_util.py
def add(a, b): return a + b

if __name__ == '__main__':
    # 仅在直接运行此文件时执行；被 import 时不执行
    print('自测:', add(1, 2))
```

```python
# main.py —— 导入
from mypkg.math_util import add      # 从包导入
from pathlib import Path             # 标准库

print(add(2, 3))

# 现代路径操作（pathlib，优于 os.path 拼接）
p = Path('data') / 'file.txt'
if p.exists():
    text = p.read_text(encoding='utf-8')

# collections 让代码更简洁
from collections import Counter, defaultdict
words = ['a', 'b', 'a', 'c', 'a']
print(Counter(words))                # Counter({'a': 3, 'b': 1, 'c': 1})
grouped = defaultdict(list)
for k, v in [('x', 1), ('x', 2), ('y', 3)]:
    grouped[k].append(v)             # 无需先判 key 是否存在

# functools.lru_cache 自动缓存（空间换时间）
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    return n if n < 2 else fib(n-1) + fib(n-2)
print(fib(100))                      # 瞬间出结果（原本指数级）
```

```bash
# 虚拟环境与依赖管理（命令行）
python -m venv .venv            # 创建虚拟环境
source .venv/bin/activate       # 激活（Windows: .venv\Scripts\activate）
pip install requests            # 装包到当前虚拟环境
pip freeze > requirements.txt   # 导出依赖清单
pip install -r requirements.txt # 别人/CI 复现依赖
deactivate                      # 退出虚拟环境

# 现代工具 uv（更快）
uv venv && uv pip install requests
```

## ⚠️ 易错点 / 面试要点

- **`__name__ == '__main__'` 的意义**：让脚本可复用（被导入无副作用）又可执行。多进程/入口脚本必备。
- **别污染全局环境**：永远先建虚拟环境再 `pip install`，否则系统 Python 被各种版本搅乱。
- **`requirements.txt` 要锁版本**：`pip freeze` 锁住确切版本，保证复现。`uv`/`poetry` 的锁文件更精确（含哈希）。
- **循环导入**：A import B、B import A 会报错。解法：重构拆分、把导入移到函数内、或用延迟导入。
- **`pathlib` > `os.path`**：现代代码用 `Path` 对象拼路径，跨平台、可读，少用字符串拼接 `/`。
- **标准库先找**：很多需求（JSON、正则、HTTP、日期）标准库就有，别急着装第三方。
- **命名别和标准库冲突**：别把文件命名成 `random.py`、`json.py`，否则 `import json` 会导入你自己的文件，导致诡异错误。
