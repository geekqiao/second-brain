# 异步与并发

> [!info] 所属：[[Python MOC]] · 上一篇 [[异常与上下文管理]] · 下一篇 [[模块包与标准库]]

## 核心概念

Python 处理「同时做多件事」有三种模型，选对场景是关键：

| 模型 | 模块 | 适合 | 并行？ |
| --- | --- | --- | --- |
| **多线程** | `threading` | IO 密集（网络/磁盘等待） | 受 GIL 限制，CPU 不真并行 |
| **多进程** | `multiprocessing` | CPU 密集（计算） | ✅ 真并行（绕过 GIL） |
| **异步** | `asyncio` | 大量 IO 并发（爬虫/服务器） | 单线程并发，极省资源 |

### GIL（全局解释器锁）
CPython 的 GIL 让**同一时刻只有一个线程执行 Python 字节码**。后果：
- 多线程**无法**加速 CPU 密集任务（甚至更慢）。
- 但对 IO 密集任务仍有用——线程等待 IO 时会让出 GIL。
- 想用多核跑计算 → 用多进程（独立解释器、独立 GIL）。

### asyncio（async/await）
- 单线程内的**协作式并发**：任务主动 `await` 让出控制权，事件循环切换。
- `async def` 定义协程；`await` 等待另一个协程；`asyncio.run()` 启动。
- **规则**：`await` 只能在 `async def` 内；调用协程函数不执行它，需 `await` 或 `create_task`。
- 配合 `asyncio.gather` 并发跑多个。

### 何时用什么
- IO 密集、并发量一般 → `threading` 或 `asyncio`。
- IO 密集、超高并发（成千上万连接）→ `asyncio`（FastAPI、aiohttp）。
- CPU 密集（数值计算、图像处理）→ `multiprocessing` 或 C 扩展。

## 代码示例

```python
# 1) 多线程：IO 密集（同时下载）
import threading, time

def fetch(url):
    time.sleep(1)            # 模拟网络 IO
    print(f'done {url}')

threads = [threading.Thread(target=fetch, args=(u,)) for u in ['a','b','c']]
for t in threads: t.start()
for t in threads: t.join()   # 总耗时约 1s（并发），而非 3s

# 2) 多进程：CPU 密集（绕过 GIL）
from multiprocessing import Pool

def square(x):
    return x * x

if __name__ == '__main__':
    with Pool(4) as p:                   # 4 个进程并行
        print(p.map(square, range(10)))  # 真正用满多核

# 3) asyncio：单线程高并发
import asyncio, time

async def fetch(url):
    await asyncio.sleep(1)    # 模拟异步 IO（非阻塞）
    return f'done {url}'

async def main():
    # 串行：3 秒
    # 并发：约 1 秒
    results = await asyncio.gather(
        fetch('a'), fetch('b'), fetch('c')
    )
    print(results)

start = time.time()
asyncio.run(main())
print(f'耗时 {time.time()-start:.2f}s')   # ~1s

# 4) 创建任务（不等立即返回）
async def main2():
    task = asyncio.create_task(fetch('a'))   # 调度但不阻塞
    # ... 做别的事
    result = await task                       # 需要时再等
```

## ⚠️ 易错点 / 面试要点

- **GIL 是高频考点**：解释「为什么多线程不能加速 CPU 计算」「为什么用多进程」。
- **`asyncio` 不是银弹**：它要求整个调用链都是异步的——一旦中间有**同步阻塞**调用（如 `time.sleep`、`requests.get`、普通 `open().read()`），会**阻塞整个事件循环**，所有协程都卡住。IO 必须用异步库（`aiohttp`、`asyncio.sleep`、`aiofiles`）。
- **调用协程 ≠ 执行协程**：`fetch('a')` 只创建协程对象，不运行；必须 `await` 或 `asyncio.create_task` + `await`，否则「协程从未被 await」警告。
- **`asyncio.run()` 是入口**：在顶层调用一次，别手动管理事件循环（旧 API 已过时）。
- **多进程开销大**：创建进程比线程贵得多（内存、启动），任务太少时反而更慢；用进程池 `Pool` 复用。
- **线程安全**：多线程共享内存需加锁（`threading.Lock`），否则数据竞争。`queue.Queue` 是线程安全的通信方式。
- **`if __name__ == '__main__'`**：Windows 下 `multiprocessing` 必须有这行（import 保护），否则无限递归 spawn。
