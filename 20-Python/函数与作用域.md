# 函数与作用域

> [!info] 所属：[[Python MOC]] · 上一篇 [[控制流（条件与循环）]] · 下一篇 [[面向对象]]

## 核心概念

函数是组织代码的基本单元。Python 的函数是**一等对象**（可赋值、传参、返回），参数机制灵活强大。理解**作用域（LEGB）** 和**参数传递模型**，能避开最常见的隐蔽 bug。

### 参数传递：传对象的引用（传共享）
Python 既非纯「传值」也非纯「传引用」，而是**传对象的引用**：
- 传入**不可变**对象（int/str/tuple）：函数内重新赋值不影响外部（类似传值）。
- 传入**可变**对象（list/dict）：函数内**原地修改**会影响外部（类似传引用）。

### 参数种类（按顺序）
1. **位置参数**：`def f(a, b)`
2. **默认参数**：`def f(a, b=10)`（默认值只求值一次 → 可变默认值陷阱，见 [[数据类型与数据结构]]）
3. **`*args`**：收集多余位置参数为 tuple。
4. **`**kwargs`**：收集多余关键字参数为 dict。
5. **仅关键字参数**：`def f(a, *, key)` —— `*` 之后必须用关键字传 `key=`。
6. **仅位置参数**（`/`）：`def f(a, b, /)` —— 之前必须用位置传。

### LEGB 作用域查找顺序
查找名字时按 **L → E → G → B**：
- **L**ocal：函数内部
- **E**nclosing：外层嵌套函数
- **G**lobal：模块全局
- **B**uilt-in：内置（`len`、`print`）
- 修改全局/外层用 `global` / `nonlocal`。

### lambda
匿名函数，单表达式，适合简短回调：`lambda x: x*2`。复杂逻辑别用。

## 代码示例

```python
# 1) 传对象的引用
def modify(lst, n):
    lst.append(99)      # 可变对象，原地修改 → 影响外部
    n = 100             # 不可变，重新赋值 → 不影响外部

data, num = [1, 2], 5
modify(data, num)
print(data, num)        # [1, 2, 99] 5

# 2) 灵活的参数
def f(a, b=10, *args, key, **kwargs):
    print(a, b, args, key, kwargs)
f(1, 2, 3, 4, key='K', extra='x')  # 1 2 (3,4) K {'extra':'x'}

# 3) 仅关键字 / 仅位置
def connect(host, port, /, *, timeout=5):  # host/port 必须位置，timeout 必须关键字
    ...
connect('localhost', 8080, timeout=10)

# 4) LEGB 作用域
x = 'global'
def outer():
    x = 'enclosing'
    def inner():
        x = 'local'
        print(x)        # local（L 优先）
    inner()
outer()

# 5) global / nonlocal 修改外层
counter = 0
def inc():
    global counter
    counter += 1        # 修改全局

def make_counter():
    count = 0
    def inner():
        nonlocal count  # 修改外层函数的 count
        count += 1
        return count
    return inner

# 6) lambda + 高阶函数
nums = [1, 2, 3]
doubled = list(map(lambda x: x * 2, nums))   # [2,4,6]
evens = list(filter(lambda x: x % 2 == 0, nums))
# 但列表推导更 Pythonic：
doubled = [x * 2 for x in nums]
```

## ⚠️ 易错点 / 面试要点

- **传可变对象的副作用**：函数内 `lst.append()` 改了外部列表。若不想改外部，传入 `copy` 或在函数内显式复制。
- **可变默认参数**（重申）：`def f(x=[])` 是经典坑，用 `None` 哨兵。
- **`global`/`nonlocal` 别滥用**：大量依赖全局变量是坏设计，优先靠参数和返回值传递状态。
- **参数顺序固定**：位置 → 默认 → `*args` → 仅关键字 → `**kwargs`，乱序会语法报错。
- **lambda 限制**：只能单表达式，不能有语句（赋值/try）。复杂逻辑用 `def`。
- **map/filter vs 推导式**：Python 社区更推崇列表/生成器推导，可读性更好。
- **闭包**：内层函数引用外层变量 = 闭包，配合 `nonlocal` 可做状态封装（见 [[装饰器迭代器与生成器]]）。
