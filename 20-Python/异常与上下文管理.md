# 异常与上下文管理

> [!info] 所属：[[Python MOC]] · 上一篇 [[装饰器迭代器与生成器]] · 下一篇 [[异步与并发]]

## 核心概念

Python 用**异常（Exception）** 处理错误，用**上下文管理器（Context Manager）** 管理资源的获取与释放。两者让代码「出错时不崩溃、用完资源必释放」。

### 异常体系
- 所有异常继承自 `BaseException`；**自定义异常应继承 `Exception`**（不是 `BaseException`，后者含 `KeyboardInterrupt`/`SystemExit`，捕获它会拦掉退出）。
- 内置常见：`ValueError`、`TypeError`、`KeyError`、`IndexError`、`FileNotFoundError`、`AttributeError`。

### try / except / else / finally
```python
try:        # 尝试执行
    ...
except SomeError as e:   # 捕获特定异常
    ...
else:       # try 没抛异常时执行（放成功后的逻辑，避免误捕获）
    ...
finally:    # 无论成败都执行（清理资源）
    ...
```

### raise
主动抛出异常：`raise ValueError('msg')`。错误信息要具体。

### 自定义异常
继承 `Exception`，按业务分类错误，便于上层针对性捕获。

### 上下文管理器（with 语句）
`with` 保证**进入时获取资源、退出时（哪怕异常）必释放**。比手写 `try/finally` 更简洁可靠。
- 实现 `__enter__` + `__exit__` 协议的对象即可用于 `with`。
- `contextlib.contextmanager` 装饰器：用生成器快速写上下文管理器。

## 代码示例

```python
# 1) 完整结构
def load(path):
    f = None
    try:
        f = open(path)
        data = f.read()
    except FileNotFoundError as e:
        print('文件不存在:', e)
        data = None
    except (PermissionError, OSError) as e:   # 多种异常一起捕获
        print('其它 IO 错误:', e)
        data = None
    else:
        print('读取成功')        # 只在没异常时执行
    finally:
        if f:
            f.close()            # 必释放
    return data

# 2) with：自动管理资源（推荐写法）
with open(path) as f:            # 退出 with 块自动 close，哪怕中途异常
    data = f.read()
# 多个资源
with open('a') as fa, open('b') as fb:
    pass

# 3) 自定义异常
class ValidationError(Exception):
    """业务校验错误"""
    pass

def set_age(age):
    if age < 0:
        raise ValidationError(f'年龄不能为负: {age}')   # 信息要具体

try:
    set_age(-1)
except ValidationError as e:
    print('校验失败:', e)

# 4) 自定义上下文管理器（两种写法）
# 写法 A：实现协议
class Timer:
    def __enter__(self):
        import time; self.start = time.time()
        return self
    def __exit__(self, exc_type, exc_val, exc_tb):
        import time
        print(f'耗时 {time.time()-self.start:.3f}s')
        return False             # 不吞异常（True 会吃掉异常）

with Timer():
    sum(range(1000000))

# 写法 B：contextmanager 装饰器（更简洁）
from contextlib import contextmanager
import time

@contextmanager
def timer():
    start = time.time()
    try:
        yield                   # with 块在此处执行
    finally:
        print(f'耗时 {time.time()-start:.3f}s')

with timer():
    sum(range(1000000))
```

## ⚠️ 易错点 / 面试要点

- **别用裸 `except:`**：`except:` 会捕获**所有**异常（含 `KeyboardInterrupt`），导致 Ctrl+C 都无效。用 `except Exception:` 或具体类型。
- **捕获要具体**：宽泛捕获会掩盖真实 bug。尽量捕获**预期的具体异常**，并记录/处理。
- **自定义异常继承 `Exception`**，不是 `BaseException`。
- **`finally` 总会执行**（即使 try 里有 return），常用于清理；但 `finally` 里再 return 会**覆盖** try 的 return，是坏味道。
- **`else` 子句的用处**：把「成功后的逻辑」放 else，避免不小心把后续代码的异常也捕获进 except。
- **资源管理用 `with`**：文件、锁、数据库连接，别手写 open/close + try/finally，`with` 更可靠（忘记 close 会导致资源泄漏）。
- **异常有性能成本**：别用异常做正常流程控制（如「捕获 KeyError 判 key 存在」→ 用 `dict.get` 或 `in`）。
