# 面向对象

> [!info] 所属：[[Python MOC]] · 上一篇 [[函数与作用域]] · 下一篇 [[装饰器迭代器与生成器]]

## 核心概念

Python 的面向对象**简洁而强大**。它支持类、继承、多态、封装，并通过**魔术方法（dunder）** 让自定义类型像内置类型一样自然工作（能用 `+`、`len()`、`for` 遍历等）。一切皆对象，类本身也是对象。

### 基础：类、属性、方法
- `class Foo:` 定义类；`__init__` 是初始化方法（不是构造器，构造器是 `__new__`）。
- `self` 指向实例本身（相当于 JS 的 this，但必须显式写）。
- 实例属性在 `__init__` 里赋值；类属性所有实例共享。

### 三大特性
- **封装**：约定 `_protected`、`__private`（名字改写，并非真私有）。
- **继承**：`class Dog(Animal):`，子类获得父类方法，可覆写。
- **多态**：不同类型对同一方法有各自实现（鸭子类型：有 `quack()` 就是鸭子）。

### 魔术方法（dunder methods）
让自定义类支持内置操作：
| 方法 | 触发 |
| --- | --- |
| `__init__` | 实例化初始化 |
| `__str__` / `__repr__` | `print()` 美化 / 调试表示 |
| `__len__` | `len(obj)` |
| `__eq__` / `__lt__` | `==` / `<` |
| `__add__` | `a + b` |
| `__getitem__` | `obj[key]` |
| `__iter__` / `__next__` | `for x in obj` |

### 鸭子类型（Duck Typing）
Python 不看重类型本身，只看重**有没有需要的方法**。「会叫会游的就是鸭子」——不强声明接口，更灵活。

### `@dataclass`（现代写法）
主要存数据的类，用 `@dataclass` 自动生成 `__init__`/`__repr__`/`__eq__`，省去样板代码。

## 代码示例

```python
# 1) 基础类 + 继承 + 多态
class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self):
        raise NotImplementedError     # 父类定义接口

class Dog(Animal):                     # 继承
    def speak(self):
        return f'{self.name}: 汪汪'    # 覆写（多态）

class Cat(Animal):
    def speak(self):
        return f'{self.name}: 喵'

for a in [Dog('旺财'), Cat('咪咪')]:
    print(a.speak())                   # 多态：同一接口不同行为

# 2) 魔术方法：让类像内置类型
class Vector:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __repr__(self):                # 调试友好
        return f'Vector({self.x}, {self.y})'
    def __add__(self, other):          # 支持 + 运算
        return Vector(self.x + other.x, self.y + other.y)
    def __eq__(self, other):           # 支持 ==
        return self.x == other.x and self.y == other.y

print(Vector(1, 2) + Vector(3, 4))     # Vector(4, 6)

# 3) @dataclass：数据类的现代写法
from dataclasses import dataclass, field

@dataclass
class User:
    name: str
    age: int
    tags: list = field(default_factory=list)   # 避免可变默认值陷阱

u = User('Ada', 30)
print(u)                               # User(name='Ada', age=30, tags=[])
# 自动生成 __init__/__repr__/__eq__，无需手写

# 4) 鸭子类型：不查类型，只要有方法就行
def make_it_speak(thing):
    return thing.speak()               # 只要 thing 有 speak() 就能跑
```

## ⚠️ 易错点 / 面试要点

- **`self` 必须显式**：方法第一个参数是实例本身（习惯叫 `self`），调用时自动传入，但定义不能省。
- **类属性 vs 实例属性**：类属性所有实例共享；在实例上赋值会**遮蔽**类属性（创建实例属性），不影响类属性本身。**可变类属性是共享陷阱**：`class C: items=[]`，所有实例共享同一个 list。
- **`__init__` 不是构造器**：真正构造是 `__new__`；`__init__` 只负责初始化已创建的实例。
- **`__str__` vs `__repr__`**：前者面向用户（`print`），后者面向开发者（调试/REPL），`__repr__` 更重要（没 `__str__` 时 fallback 到它）。
- **私有是约定不是强制**：`__x` 做了名字改写（`_ClassName__x`）但仍可访问；`_x` 纯约定。Python 没有真正的访问控制。
- **MRO 与多继承**：Python 支持多继承，方法解析顺序（MRO）用 C3 线性化，复杂继承看 `Cls.mro()`。菱形继承要小心。
- **优先组合而非继承**：深继承树难维护，「has-a」（组合）往往比「is-a」（继承）更灵活。
