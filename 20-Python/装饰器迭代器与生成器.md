# 装饰器、迭代器与生成器

> [!info] 所属：[[Python MOC]] · 上一篇 [[面向对象]] · 下一篇 [[异常与上下文管理]]

## 核心概念

这三个特性是 Python 「进阶」的核心，也是写出**地道（Pythonic）** 代码的关键。它们都建立在一个基础上：**函数是一等对象**，且**一切皆可迭代**。

### 装饰器（Decorator）
**在不修改原函数的前提下，给函数增加功能**。本质：装饰器是一个**接收函数、返回函数**的函数。`@dec` 是语法糖，等价于 `func = dec(func)`。
- 用途：日志、计时、权限校验、缓存、注册路由（Flask/FastAPI）。

### 迭代器（Iterator）
实现了 `__iter__` 和 `__next__` 的对象。`for` 循环的本质就是不断调用 `next()` 直到 `StopIteration`。
- 可迭代（Iterable）≠ 迭代器（Iterator）：list 是可迭代的，但 `iter(lst)` 才是迭代器。

### 生成器（Generator）
**用函数创建迭代器的简便方式**：函数里有 `yield`，就变成生成器。每次 `next` 执行到 `yield` 暂停并返回值，下次从暂停处继续。
- **惰性求值**：按需产出，不一次性生成全部，**省内存**（处理大文件/无限序列）。
- 生成器表达式：`(x*2 for x in range(10))`，类似列表推导但惰性。

## 代码示例

```python
import time
from functools import wraps

# 1) 装饰器：计时
def timer(func):
    @wraps(func)                # 保留原函数名/文档
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f'{func.__name__} 耗时 {time.time()-start:.4f}s')
        return result
    return wrapper

@timer                          # 等价于 slow = timer(slow)
def slow():
    time.sleep(0.5)
    return 'done'
slow()                          # 打印耗时

# 2) 带参数的装饰器（三层嵌套）
def repeat(n):
    def deco(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for _ in range(n):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return deco

@repeat(3)                      # 调用三次
def greet(): print('hi')

# 3) 生成器：惰性序列
def counter(max):
    n = 0
    while n < max:
        yield n                # 产出并暂停
        n += 1

for x in counter(3):           # 0 1 2，不预先生成大列表
    print(x)

# 处理大文件，逐行产出，内存恒定
def read_large(path):
    with open(path) as f:
        for line in f:
            yield line.strip()

# 4) 生成器表达式 vs 列表推导
squares_list = [x**2 for x in range(1000000)]   # 立即占内存
squares_gen  = (x**2 for x in range(1000000))   # 惰性，几乎不占内存
print(sum(x**2 for x in range(10)))             # 直接喂给 sum，无需中间列表

# 5) 自定义迭代器（实现协议）
class CountDown:
    def __init__(self, start): self.n = start
    def __iter__(self): return self
    def __next__(self):
        if self.n <= 0: raise StopIteration
        self.n -= 1
        return self.n + 1

for x in CountDown(3): print(x)   # 3 2 1
```

## ⚠️ 易错点 / 面试要点

- **装饰器用 `@wraps`**：不加 `@wraps(func)`，被装饰函数的 `__name__`/`__doc__` 会变成 wrapper 的，影响调试与反射。
- **`@deco` 是语法糖**：`@timer` 等价于 `func = timer(func)`，在**定义时**就执行装饰。带参装饰器是三层函数。
- **生成器只能迭代一次**：耗尽后不会再产出（除非重建）。需要多次遍历就转 list，或用 `itertools.tee`。
- **`yield` 让函数变生成器**：哪怕函数体里有 `yield`，调用时**不执行函数体**，只返回生成器对象——这是新手最意外的点。
- **惰性求值省内存**：处理大文件/大数据，用生成器逐行/逐块，别 `read()` 整个吞进内存。
- **`return` 在生成器里**：触发生成器结束（相当于 `raise StopIteration(value)`），不可在生成器内 `return` 一个非 None 的值（Python 3.3+ 允许但语义是停止）。
- **可迭代 vs 迭代器**：`for` 能遍历的是 Iterable（有 `__iter__`）；能 `next()` 的是 Iterator。`for x in lst` 内部先 `iter(lst)` 得到迭代器。
