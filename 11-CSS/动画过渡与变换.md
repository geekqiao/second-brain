# 动画、过渡与变换

> [!info] 所属：[[CSS MOC]] · 上一篇 [[响应式设计]] · 下一篇 [[现代 CSS 特性]]

## 核心概念

CSS 让界面「动起来」有三种主力武器，各有适用场景：

| 技术 | 触发 | 适用 |
| --- | --- | --- |
| **transition 过渡** | 状态变化（hover/focus/class 切换） | 简单的「A→B」平滑变化 |
| **transform 变换** | 任何时候 | 平移/缩放/旋转/倾斜（不触发布局，性能好） |
| **@keyframes 动画** | 自动播放 | 复杂、多步骤、循环的动画 |

### transition
`transition: <属性> <时长> <缓动函数> <延迟>`
- 只对**能插值**的属性有效（颜色、长度、透明度等；`display` 不行）。
- 常用缓动：`ease`（默认）、`ease-in/out`、`linear`、`cubic-bezier(...)`。

### transform（不触发布局重排）
- `translate(x, y)` / `translateX/Y`：位移。
- `scale(n)`：缩放。
- `rotate(deg)`：旋转。
- `skew(deg)`：倾斜。
- 可组合：`transform: translate(10px,0) rotate(45deg)`。

### @keyframes + animation
定义关键帧百分比，再用 `animation` 绑定：`name duration timing-function delay iteration-count direction ...`。

### 性能要点
浏览器渲染分两步：**Layout（布局）→ Paint（绘制）→ Composite（合成）**。
- 改 `width/top/margin` 触发 **Layout**，最贵。
- 改 `transform` / `opacity` 只触发 **Composite**，**最便宜**，能跑 60fps。
- 动画优先用 `transform` 和 `opacity`；需要时用 `will-change: transform` 提示浏览器优化。

## 代码示例

```css
/* transition：hover 时平滑变色 + 上浮 */
.btn {
  background: #2563eb;
  transition: background .2s ease, transform .2s ease;
}
.btn:hover {
  background: #1d4ed8;
  transform: translateY(-2px);   /* 上浮，不触发布局 */
}

/* transform：悬停放大 */
.card { transition: transform .3s; }
.card:hover { transform: scale(1.05); }

/* @keyframes：加载旋转动画 */
@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}
.loader {
  width: 40px; height: 40px;
  border: 4px solid #eee;
  border-top-color: #2563eb;
  border-radius: 50%;
  animation: spin .8s linear infinite;  /* 无限循环 */
}

/* 多步关键帧：弹跳进入 */
@keyframes pop-in {
  0%   { transform: scale(0); opacity: 0; }
  70%  { transform: scale(1.1); opacity: 1; }
  100% { transform: scale(1); }
}
.modal { animation: pop-in .4s ease both; }

/* 尊重用户偏好：减少动效 */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: .01ms !important;
    transition-duration: .01ms !important;
  }
}
```

## ⚠️ 易错点 / 面试要点

- **动画 `transform/opacity` 优先**：用 `top/left/width/margin` 做动画会触发重排，卡顿掉帧。位移永远用 `translate`。
- **`display` 不能 transition**：从 `none` 到 `block` 不会渐变。常见方案：用 `opacity`+`visibility` 配合，或用 `transform: scale(0)`。
- **`transition` 要写在「基础态」而非「结束态」**：写在 `:hover` 里，鼠标移出会瞬间跳回（没有过渡）。
- **`auto` 值无法过渡**：从 `height: 0` 到 `height: auto` 不平滑，需用 `max-height` 技巧或 JS 测量。
- **性能要测**：DevTools → Animations / Performance 面板看帧率，掉帧就回归 `transform/opacity`。
- **`prefers-reduced-motion`**：照顾前庭功能敏感的用户，给他们关闭动画的选项，是无障碍必备（见 [[可访问性 a11y]]）。
