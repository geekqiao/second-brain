# 选择器与优先级

> [!info] 所属：[[CSS MOC]] · 下一篇 [[盒模型]]

## 核心概念

CSS 的「层叠（cascade）」回答一个问题：**当多条规则都指向同一元素、属性冲突时，谁赢？** 答案由三件事决定：**优先级（specificity）→ 来源与顺序 → `!important`**。

### 选择器家族
| 类型 | 示例 | 说明 |
| --- | --- | --- |
| 元素（类型） | `div`, `p` | 选标签 |
| 类 class | `.btn` | 最常用，可复用 |
| ID | `#header` | 页面唯一 |
| 属性 | `[type="text"]` | 按属性选 |
| 通配 | `*` | 选所有（慎用，性能/影响大） |
| 后代 | `.nav a` | `.nav` 里的所有 `a`（含深层） |
| 直接子代 | `.nav > a` | 仅直接子级 |
| 伪类 | `:hover`, `:focus`, `:nth-child(n)` | 元素的**状态**/位置 |
| 伪元素 | `::before`, `::after` | 生成**不存在的内容** |
| 组合 | `a, button` | 多选 |

### 优先级计算（specificity）
用 `(a, b, c, d)` 四位数表示，从左到右比较，左边大的整体大：
- `a`：`!important`（最高，但请少用）
- `b`：内联样式 `style="..."`
- `c`：ID 数量
- `d`：class / 属性 / 伪类 数量
- `e`：元素 / 伪元素 数量

> 习惯上记为 `(ID, class, element)` 三档。例：`#nav .item a` = (1,1,1)。

### 继承（inheritance）
- 某些属性会从父元素**自动继承**给后代：`color`、`font-*`、`line-height`、`text-*` 等。
- 盒模型相关（`border`/`margin`/`padding`/`width`）**不继承**。
- 可用 `inherit` / `initial` / `unset` / `revert` 显式控制。

## 代码示例

```css
/* 优先级：(0,1,1) 一个 class + 一个元素 */
.card h2 { color: darkblue; }

/* 优先级：(1,0,0) 一个 ID —— 比上面高，胜出 */
#title { color: red; }

/* 通配与组合 */
* { box-sizing: border-box; }        /* 全局盒模型修正 */
nav > a { color: inherit; }          /* 直接子链接 */

/* 伪类：状态与位置 */
a:hover { text-decoration: underline; }
li:nth-child(odd) { background: #f5f5f5; } /* 奇数行 */
input:focus-visible { outline: 2px solid blue; }

/* 伪元素：生成内容 */
.required::after { content: " *"; color: red; } /* 必填标记 */

/* 继承控制 */
.parent { color: #333; }
.child { color: inherit; }   /* 显式继承（即便默认不继承的属性也能继承） */
```

```html
<h1 id="title" class="card">优先级演示</h1>
<!-- #title (1,0,0) 胜过 .card h2 (0,1,1)，显示红色 -->
```

## ⚠️ 易错点 / 面试要点

- **`!important` 是坏味道**：它打破层叠规则，让样式无法被正常覆盖，导致维护噩梦。优先靠提升选择器优先级来解决，而非 `!important`。
- **ID 选择器优先级太高**：一旦用 `#id`，后面很难覆盖，所以现代实践倾向少用 ID 写样式，多用 class。
- **内联样式最该避免**：`style="..."` 优先级仅次于 `!important`，且把结构与表现混在一起，难以复用。
- **顺序也重要**：优先级相同时，**后写的覆盖先写的**。这就是「层叠」。
- **`*{margin:0}` 慎用通配重置**：影响范围大、可能误伤，用现代 reset/normalize 更稳妥。
- **伪类 `:` vs 伪元素 `::`**：单冒号是状态（`:hover`），双冒号是生成内容（`::before`），现代规范建议区分书写（旧代码伪元素也写单冒号仍兼容）。
