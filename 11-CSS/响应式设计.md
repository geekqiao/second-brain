# 响应式设计

> [!info] 所属：[[CSS MOC]] · 上一篇 [[Grid 布局]] · 下一篇 [[动画过渡与变换]]

## 核心概念

**响应式设计（Responsive Design）** 让同一套页面在不同尺寸的设备（手机、平板、桌面）上都好用。核心三件套：**流式布局 + 弹性单位 + 媒体查询**。配合移动优先（mobile-first）的书写策略，代码更简洁。

### 媒体查询（Media Query）
`@media (条件) { ... }`，根据视口宽度等条件应用不同样式。

> **移动优先**：默认写小屏样式（基础），再用 `min-width` 向大屏追加。逻辑是「从小到大增强」，比「从大到小覆盖」更省代码、更符合渐进增强思想。

### 尺寸单位
| 单位 | 基准 | 何时用 |
| --- | --- | --- |
| `px` | 绝对像素 | 边框、固定间距 |
| `%` | 父元素 | 流式宽度 |
| `rem` | 根字号（html） | 字号、间距（随用户放大） |
| `em` | 父/自身字号 | 组件内相对缩放（注意会层层乘） |
| `vw` / `vh` | 视口宽/高 1% | 全屏区域 |
| `vmin` / `vmax` | 视口较小/较大边 | 自适应正方形等 |
| `clamp(min, ideal, max)` | 区间限制 | 流体字号 `clamp(1rem, 4vw, 2rem)` |

### 响应式图片（HTML 侧）
`<img srcset>` 让浏览器按屏幕选图，详见 [[多媒体与嵌入]]。

### 容器查询（Container Queries）—— 新方向
媒体查询按**视口**变化；容器查询按**组件父容器**变化，让组件真正自适应它所在的任意位置（见 [[现代 CSS 特性]]）。

## 代码示例

```css
/* 移动优先：默认（小屏）样式 */
.grid {
  display: grid;
  gap: 1rem;
  grid-template-columns: 1fr;   /* 手机：单列 */
}

/* ≥600px：两列（在基础之上追加，不覆盖重写） */
@media (min-width: 600px) {
  .grid { grid-template-columns: 1fr 1fr; }
}

/* ≥900px：三列 */
@media (min-width: 900px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

/* 流体字号：不小于 1rem，不大于 2rem，中间随视口缩放 */
h1 { font-size: clamp(1rem, 5vw, 2rem); }

/* rem 做间距：尊重用户在浏览器设置的根字号 */
body { padding: 1.5rem; line-height: 1.6; }

/* 用 dpr / 暗色模式做增强 */
@media (prefers-color-scheme: dark) {
  body { background: #1a1a1a; color: #eee; }
}
@media (prefers-reduced-motion: reduce) {
  * { animation: none; transition: none; } /* 尊重「减少动效」无障碍偏好 */
}
```

```html
<!-- 别忘了这行，否则手机上会被当成桌面缩放显示 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

## ⚠️ 易错点 / 面试要点

- **viewport meta 不可少**：没有它，手机会把桌面宽度（如 980px）硬塞进屏幕，响应式 CSS 全部失效。放在 `<head>` 最前。
- **用 `min-width`（移动优先）而非 `max-width`**：默认小屏、向大屏增强，避免大量覆盖代码。
- **断点看内容而非设备**：不要写「iPhone 断点」「iPad 断点」，而应在**布局开始变难看**的宽度设断点。常用断点 ≈ 600/900/1200px。
- **别用 `px` 写字号**：用 `rem`，让用户调整浏览器根字号时页面跟着放大（无障碍）。
- **`em` 会嵌套乘**：父 1.2em、子再 1.2em 会变成 1.44 倍，容易失控。组件外层用 rem 更稳。
- **测试要转屏**：DevTools 设备模拟器横竖屏都要测。
- **响应式 ≠ 缩小**：小屏不是把桌面版等比缩小，而是**重排**（隐藏次要内容、单列化、放大触控目标）。
