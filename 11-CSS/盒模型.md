# 盒模型

> [!info] 所属：[[CSS MOC]] · 上一篇 [[选择器与优先级]] · 下一篇 [[Flexbox 布局]]

## 核心概念

浏览器把**每个元素都当成一个矩形盒子**。理解盒子由内到外的四层，是所有布局的基础。

### 盒子的四层（从内到外）
`content（内容）→ padding（内边距）→ border（边框）→ margin（外边距）`

```
┌──────────────── margin（外）────────────────┐
│  ┌──────────── border（边框）────────────┐  │
│  │  ┌──────── padding（内）──────────┐  │  │
│  │  │      content（内容区）          │  │  │
│  │  └────────────────────────────────┘  │  │
│  └──────────────────────────────────────┘  │
└────────────────────────────────────────────┘
```

### `box-sizing`：两种模型
- **`content-box`（默认）**：你设的 `width` 只是**内容区**。加 padding/border 后，盒子实际变胖 → 容易溢出、布局算不准。
- **`border-box`**：`width` 包含 **content + padding + border**。设多少就是多少，**直觉友好**。

> 几乎所有现代项目都在全局加 `* { box-sizing: border-box; }`。

### margin 折叠（collapse）
相邻的**垂直**外边距会合并（取较大值，而非相加），出现在：相邻兄弟、父与首个/末个子元素、空块自身。**只有垂直、只有普通文档流**才折叠；Flex/Grid 容器内的元素**不会折叠**。

### padding vs margin 何时用
- `padding`：盒子的**内部**留白，属于盒子背景区（会显示背景色/边框）。
- `margin`：盒子**外部**的间距，推开邻居，不显示自身背景。

## 代码示例

```css
/* 全局：统一为 border-box，避免布局意外 */
*, *::before, *::after {
  box-sizing: border-box;
}

/* content-box（默认）：设 200px 但实际 240px（加 padding+border） */
.box-content {
  box-sizing: content-box;
  width: 200px;
  padding: 10px;
  border: 5px solid #333;
  /* 实际宽度 = 200 + 10*2 + 5*2 = 230px */
}

/* border-box：设 200px 就是 200px，padding/border 向内挤 */
.box-border {
  box-sizing: border-box;
  width: 200px;
  padding: 10px;
  border: 5px solid #333;
  /* 实际宽度仍是 200px，内容区被压缩 */
}

/* margin 折叠演示：两个块间距是 30px，不是 50px */
.top    { margin-bottom: 30px; }
.bottom { margin-top: 50px; }
/* 间距 = max(30, 50) = 50px（大的吞小的） */

/* Flex 容器内不折叠：间距 80px 相加 */
.flex-parent { display: flex; flex-direction: column; }
```

```html
<div class="box-border">border-box：所见即所得</div>
```

## ⚠️ 易错点 / 面试要点

- **默认 content-box 是万恶之源**：忘了 `box-sizing: border-box`，加 padding 后布局错位是新手最常见翻车。务必全局重置。
- **margin 折叠反直觉**：两个垂直相邻块的 margin 不相加而取大值，常让人困惑「为什么间距不对」。
- **margin 不会撑大盒子**，但可能影响外部布局；负 margin 可用于重叠效果但要小心。
- **行内元素（span/a）的 padding/margin 垂直方向不占空间**：只有水平方向生效，垂直会被忽略（不影响行高）。需要垂直 padding 就改成 `inline-block` 或 `block`。
- **百分比 padding/margin**：基准不同——padding 的百分比基于**宽度**（即使设的是垂直 padding），margin 同理。这是为了保持宽高比例一致。
- **宽度未知的水平居中**：块级元素定宽用 `margin: 0 auto`；不定宽用 Flexbox（见 [[Flexbox 布局]]）。
