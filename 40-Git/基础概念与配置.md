# 基础概念与配置

> [!info] 所属：[[Git MOC]] · 下一篇 [[基本工作流]]

## 核心概念

学 Git 第一步不是背命令，而是建立**「三个区 + 提交链」**的心智模型。模型清晰了，所有命令都是对它的操作，排错也有据可循。

### 三个区（核心心智模型）
```
工作区 (Working Directory)
   │  git add ↓        ↑ git restore
暂存区 (Staging Area / Index)
   │  git commit ↓     ↑ git checkout
版本库 (Repository, .git)
```
- **工作区**：你直接编辑的文件（肉眼看到的）。
- **暂存区**：`git add` 进来、**准备好要提交**的快照（可挑选，不是全部改动）。
- **版本库**：`git commit` 后正式记录的历史，存于 `.git`。

> 关键：commit 只记录**暂存区**的内容，不是工作区。所以流程永远是 `add`（挑选）→ `commit`（固化）。

### 提交与历史
- 每次提交生成一个唯一 **commit hash**（如 `a1b2c3d`）。
- 提交通过**父提交**串成链 → 这就是历史。**分支**只是指向某个提交的**可动指针**。

### 仓库初始化
- `git init`：把当前目录变成 Git 仓库（生成 `.git`）。
- `git clone <url>`：从远程复制一份完整仓库（含全部历史）。

### 全局配置（一次性）
- `git config --global user.name/email`：你的身份（写进每次提交）。
- 配置存于 `~/.gitconfig`。

### `.gitignore`
告诉 Git **哪些文件不要跟踪**（如 `node_modules/`、`.env`、`build/`、`.venv/`）。已跟踪的文件加进 ignore 不生效，要先取消跟踪。

## 代码示例（命令）

```bash
# 全局配置（每台机器做一次）
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main     # 默认分支名改 main
git config --global core.editor "code --wait"   # 设默认编辑器
git config --global --list                       # 查看所有配置

# 初始化 / 克隆
git init                      # 当前目录变成仓库
git clone https://github.com/x/y.git   # 克隆远程仓库
git clone --depth 1 <url>     # 浅克隆（只取最新，省时间/流量）

# .gitignore 示例
cat > .gitignore <<'EOF'
# 依赖
node_modules/
.venv/
__pycache__/

# 构建产物
dist/
build/

# 敏感/环境
.env
*.local

# 系统
.DS_Store
EOF

# 查看仓库状态（最常用，养成随时敲的习惯）
git status
```

## ⚠️ 易错点 / 面试要点

- **commit 记录的是暂存区，不是工作区**：改了文件但没 `add`，`commit` 不会包含它。新手常困惑「明明改了，怎么没提交」。
- **`.gitignore` 不追溯**：已被 Git 跟踪的文件，后加进 `.gitignore` 不会停止跟踪。要先 `git rm --cached <file>` 取消跟踪再提交。
- **别提交敏感信息**：`.env`、密钥、密码一旦提交进历史，即使后面删除，历史里仍能翻出。提交前检查 `.gitignore`；若已提交需用 `git filter-repo` 清理历史并视作泄露处理（轮换密钥）。
- **`git config` 区分级别**：`--global`（用户级，存 `~/.gitconfig`）vs 项目级（仓库 `.git/config`，不加 `--global`）。项目级覆盖全局。
- **身份配置很重要**：`user.email` 决定提交与 GitHub 账号的关联（贡献图绿格子）。用错邮箱会导致提交不计入你的 GitHub。
- **`.git` 目录是命根子**：所有历史在里面，删了就没了（除非有远程）。别手动改它。
- **克隆即完整**：`clone` 得到全部历史，离线也能看 log、切分支（分布式的优势）。
