# 类型与类型转换

> [!info] 所属：[[JavaScript MOC]] · 下一篇 [[作用域与闭包]]

## 核心概念

JS 的类型系统是面试与 bug 的高发区。理解它要抓住两件事：**原始类型 vs 引用类型**，以及**显式 vs 隐式类型转换**。

### 7 种原始类型（Primitive）+ 对象
| 原始类型 | 示例 | 备注 |
| --- | --- | --- |
| `number` | `1`, `3.14` | 统一浮点（无 int/float 之分） |
| `string` | `'hi'` | 不可变 |
| `boolean` | `true` | |
| `null` | `null` | 「空值」，`typeof` 返回 `'object'`（历史 bug） |
| `undefined` | `undefined` | 未赋值 |
| `symbol` | `Symbol()` | 唯一标识 |
| `bigint` | `123n` | 大整数 |
- **对象 `object`**（含数组、函数、日期等）是引用类型，可变。

### 原始 vs 引用（赋值的本质）
- **原始**：按**值**赋值/传递。复制是独立副本。
- **引用**：按**引用**（内存地址）传递。复制只是复制地址，两个变量指向同一对象，改一个另一个也变。

### `==` vs `===`
- `===`（严格相等）：类型不同直接 `false`，不转换。
- `==`（宽松相等）：类型不同先**隐式转换**再比，规则复杂易错。
> **规则：永远用 `===`，除非你有明确理由用 `==`。** 判 `null/undefined` 双等于可用 `x == null`（同时匹配两者）。

### 隐式转换的几个「名场面」
- `+` 优先字符串：`1 + '2'` → `'12'`；但 `1 - '2'` → `-1`（转数字）。
- `[] + []` → `''`；`[] + {}` → `'[object Object]'`；`{} + []` → `0`（视上下文）。
- `if` / 三元 / `&&`/`||` 判断时会转 boolean。

### Truthy / Falsy
- **Falsy**（6 个）：`false`、`0`、`''`、`null`、`undefined`、`NaN`（ES2020 加 `0n`）。
- 其余都是 **truthy**（包括 `[]`、`{}`、`'0'`、`'false'`！）。

## 代码示例

```js
// 原始 vs 引用
let a = 1, b = a; b = 2;          // a 仍是 1（值复制）
let o1 = { n: 1 }, o2 = o1; o2.n = 2;  // o1.n 也变 2（共享引用）

// 浅拷贝 vs 深拷贝
const shallow = { ...o1 };        // 展开只复制一层
const deep = structuredClone(o1); // 深拷贝（现代 API）

// 永远用 ===
0 === false   // false（类型不同）
0 == false    // true（隐式转换，坑）
'' == 0       // true（坑）
null == undefined  // true（唯一有用的 ==）

// 显式转换（推荐，可读）
Number('42')      // 42
String(42)        // '42'
Boolean(0)        // false
parseInt('12px')  // 12

// + 的字符串优先
1 + 2 + '3'       // '33'（先 1+2=3，再 '3'→'33'）
'1' + 2 + 3       // '123'（从左到右都变字符串）

// 判 NaN 用 Number.isNaN，别用全局 isNaN
Number.isNaN(NaN) // true
isNaN('abc')      // true（全局 isNaN 会先转数字，坑）
```

## ⚠️ 易错点 / 面试要点

- **`typeof null === 'object'`**：历史遗留 bug，判 null 用 `x === null`。
- **`typeof` 区分数组不够**：`typeof [] === 'object'`，判数组用 `Array.isArray([])`。
- **`0.1 + 0.2 !== 0.3`**：浮点精度问题（所有 IEEE754 语言通病），用容差比较 `Math.abs(a-b) < 1e-9`。
- **`==` 的转换规则**：背不下来就别用，统一 `===`。
- **`{}` 在行首会被当块**：`{}` + 语句可能解析成代码块而非对象字面量。
- **`'0'` 和 `[]` 是 truthy**：别用 `if (maybeArray)` 判断「有没有元素」，要用 `.length`。
- **浅拷贝陷阱**：`Object.assign` / 展开只复制一层，嵌套对象仍共享引用。
