# 模块化与包管理

> [!info] 所属：[[JavaScript MOC]] · 上一篇 [[DOM 与事件]]

## 核心概念

**模块化**把代码拆成独立、可复用的文件，解决全局污染、依赖管理、复用问题。JS 历经多种模块方案，如今 **ES Modules（ESM）** 是浏览器与 Node 的统一标准。**npm** 是 JS 世界的包管理器，拥有全球最大的开源生态。

### 模块方案演进
| 方案 | 环境 | 现状 |
| --- | --- | --- |
| IIFE | 浏览器 | 古老，已淘汰（见 [[作用域与闭包]]） |
| CommonJS（`require`） | Node | Node 传统，服务端老代码 |
| AMD（`define`） | 浏览器 | 已淘汰 |
| **ES Modules（`import/export`）** | 浏览器 + Node | **现代标准，统一方案** |

### ESM 语法
- `export`：导出（命名导出 / 默认导出）。
- `import`：导入。
- **静态分析**：`import` 在顶层、编译时确定依赖（便于 tree-shaking 摇掉无用代码）。
- 动态导入：`import('./mod')` 返回 Promise，用于按需加载。

### npm 与包管理
- `package.json`：项目清单（依赖、脚本、元信息）。
- `npm install`：装依赖到 `node_modules`。
- `npm run <script>`：跑 `scripts` 里定义的命令。
- 替代品：`pnpm`（更快、省磁盘）、`yarn`。
- **语义化版本（SemVer）**：`major.minor.patch`，`^1.2.3` 允许 1.x，`~1.2.3` 允许 1.2.x。

### 构建工具（为什么需要）
浏览器原生支持 ESM，但实际项目为**性能**（合并、压缩、tree-shaking、转译 TS/JSX）会用打包器：**Vite**（现代首选，开发用原生 ESM 极快）、**Webpack**（传统霸主）。

## 代码示例

```js
// math.js —— 导出
export const add = (a, b) => a + b;        // 命名导出
export function mul(a, b) { return a * b; }
export default function sub(a, b) { return a - b; }  // 默认导出（每模块一个）

// app.js —— 导入
import sub, { add, mul } from './math.js'; // 默认不用{}，命名用{}
import * as math from './math.js';          // 全部导入为一个对象
const diff = sub(5, 3);

// 动态导入（按需加载，返回 Promise）
button.addEventListener('click', async () => {
  const { heavyFn } = await import('./heavy.js');  // 点了才加载
  heavyFn();
});
```

```json
// package.json
{
  "name": "my-app",
  "version": "1.0.0",
  "type": "module",            // 声明使用 ESM（.js 当 ESM 解析）
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "test": "vitest"
  },
  "dependencies": {
    "express": "^4.18.0"        // ^ 允许 4.x 自动升级
  },
  "devDependencies": {
    "vite": "^5.0.0"
  }
}
```

```bash
# 包管理常用命令（见 [[Linux MOC]] / [[Git MOC]] 协作流程）
npm install            # 装 package.json 里所有依赖
npm install axios      # 装单个包并写入 dependencies
npm install -D vitest  # 装为开发依赖（devDependencies）
npm run dev            # 跑 scripts.dev
npx create-vite my-app # 用 npx 运行一次性命令
```

## ⚠️ 易错点 / 面试要点

- **默认导出 vs 命名导出**：默认导出导入时不加 `{}`、名字可任意；命名导出必须加 `{}` 且名字要对应。混用易错。
- **CommonJS 与 ESM 互操作有坑**：`require` 不能直接拿 ESM 的命名导出，尽量项目内统一一种（现代用 ESM）。
- **`node_modules` 巨大、别提交 Git**：`.gitignore` 必须排除它，靠 `package.json` + `package-lock.json` 复现依赖。
- **`dependencies` vs `devDependencies`**：前者运行时需要（如 react、express），后者仅开发需要（如 vite、测试库）。生产部署通常只装 dependencies。
- **版本锁文件**：`package-lock.json`（npm）/ `pnpm-lock.yaml` 锁定确切版本，保证团队/CI 装出一致结果，**必须提交**。
- **`^` vs `~`**：`^1.2.3` 接受 1.x.x（小版本升级），`~1.2.3` 只接受 1.2.x。前者更新更积极，偶尔引入破坏。
- **tree-shaking 依赖 ESM 静态分析**：用 `export` 命名导出，打包器才能摇掉未用代码；CommonJS 的 `require` 无法 tree-shake。
