# 作用域与闭包

> [!info] 所属：[[JavaScript MOC]] · 上一篇 [[类型与类型转换]] · 下一篇 [[原型与 this]]

## 核心概念

**作用域（scope）** 决定变量在何处可见；**闭包（closure）** 是函数「记住」它出生时所在作用域的能力。理解二者，才能看懂模块、回调、hooks 等一切现代 JS 模式。

### 三种声明与作用域
| 声明 | 作用域 | 提升 | 可重赋值 | 重复声明 |
| --- | --- | --- | --- | --- |
| `var` | **函数**作用域 | 提升（值为 `undefined`） | ✓ | ✓ |
| `let` | **块**作用域 `{}` | 暂时性死区（TDZ） | ✓ | ✗ |
| `const` | 块作用域 | TDZ | ✗（但对象内部可改） | ✗ |

> **规则：默认 `const`，需重赋值才 `let`，永远别用 `var`。**

### 提升（Hoisting）
- `var` 声明被提升到作用域顶部，但赋值留下，所以「提前用」得到 `undefined`。
- `function` 声明整体提升（连函数体），可提前调用。
- `let/const` 也有提升，但在声明前处于**暂时性死区（TDZ）**，访问直接报错。

### 作用域链
函数嵌套时，内层找不到变量就沿**词法作用域**（书写时的位置）向外层找，直到全局。

### 闭包（Closure）
**函数 + 它引用的外层变量**的组合。即使外层函数已返回，内层函数仍能访问那些变量——它们未被回收。
- 用途：数据私有化、工厂函数、回调状态、防抖/节流、模块模式。

### IIFE（立即调用函数表达式）
`(function(){ ... })()`，经典的作用域隔离方式（ES Modules 前的主流模块模式）。

## 代码示例

```js
// 1) var 的坑 vs let 的块作用域
function demo() {
  if (true) {
    var x = 1;     // 函数作用域，泄漏到函数内
    let y = 2;     // 块作用域，只在 if 内
  }
  console.log(x);  // 1
  console.log(y);  // ReferenceError
}

// 2) 暂时性死区 TDZ
console.log(z); // ReferenceError（不是 undefined）
let z = 5;

// 3) 经典闭包：计数器（数据私有）
function createCounter() {
  let count = 0;                    // 外层变量
  return function () {              // 内层函数闭包住 count
    count++;
    return count;
  };
}
const c = createCounter();
c(); c(); // 2。count 从外部无法直接访问，实现了「私有变量」

// 4) 闭包陷阱：循环中的 var（经典面试题）
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);  // 输出 3 3 3（共享同一个 i）
}
// 修复①：let（每轮新的块作用域）
for (let i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);  // 0 1 2
}
// 修复②：IIFE 捕获当时的 i
for (var i = 0; i < 3; i++) {
  ((j) => setTimeout(() => console.log(j), 0))(i);
}

// 5) 工厂函数
function multiplier(factor) {
  return (n) => n * factor;   // 闭包记住 factor
}
const double = multiplier(2);
double(5); // 10
```

## ⚠️ 易错点 / 面试要点

- **闭包循环陷阱**（输出 3 3 3）：根因是 `var` 共享一个函数级变量；`let` 每轮建新块作用域即解决。理解这个题 = 理解闭包 + 作用域。
- **`const` 对象仍可改内部**：`const obj = {}` 后 `obj.a = 1` 合法（const 锁的是绑定，不是值）。要完全冻结用 `Object.freeze`。
- **TDZ 报错**：`let/const` 声明前访问会抛 `ReferenceError`，不像 `var` 给 `undefined`。
- **闭包导致内存不释放**：若闭包长期持有大对象引用，会阻止 GC。不需要时置 `null`。
- **词法作用域 vs this**：作用域是**书写时**确定的（静态），`this` 是**调用时**确定的（动态），别混淆（见 [[原型与 this]]）。
- **模块即闭包**：ES Module 的导出变量本质就是模块作用域内的闭包变量，对外只暴露引用。
