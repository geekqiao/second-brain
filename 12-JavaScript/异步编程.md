# 异步编程

> [!info] 所属：[[JavaScript MOC]] · 上一篇 [[原型与 this]] · 下一篇 [[ES6+ 核心特性]]

## 核心概念

JS 是**单线程**的（一次只做一件事）。为不阻塞 UI，它用**事件循环（Event Loop）** 处理异步：把耗时任务（网络、定时器、IO）交给宿主环境，完成后排进队列，主线程空闲时再回头执行回调。

### Event Loop 机制
1. 主线程跑**同步代码**（调用栈）。
2. 遇到异步任务，交给 Web API（浏览器）/ Node API 处理，**不阻塞**。
3. 完成后回调进入**任务队列**。
4. 栈清空后，事件循环把队列里的任务推入栈执行。

### 宏任务（Macro）vs 微任务（Micro）
| 类型 | 示例 | 优先级 |
| --- | --- | --- |
| **微任务** | `Promise.then`、`queueMicrotask`、`MutationObserver` | 高（每轮清空所有微任务） |
| **宏任务** | `setTimeout`、`setInterval`、`setImmediate`、IO、UI 渲染 | 低（每轮取一个） |

> 口诀：**同步 → 微任务全部 → 一个宏任务 → 渲染 → 微任务…**

### 异步演进三阶段
1. **回调（Callback）**：原始方式，层层嵌套→「回调地狱」。
2. **Promise**：用链式 `.then()` 解决嵌套，三种状态 pending/fulfilled/rejected。
3. **async/await**：Promise 的语法糖，用同步写法写异步，可 try/catch。

## 代码示例

```js
// 1) 执行顺序（高频面试题）
console.log('1 同步');
setTimeout(() => console.log('4 宏任务'), 0);
Promise.resolve().then(() => console.log('3 微任务'));
console.log('2 同步');
// 输出：1 → 2 → 3 → 4

// 2) Promise 封装异步
function delay(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}
delay(500).then(() => console.log('半秒后'));

// 3) Promise 链：解决回调地狱
fetch('/api/user')
  .then(res => res.json())
  .then(user => fetch(`/api/posts?uid=${user.id}`))
  .then(res => res.json())
  .catch(err => console.error('任一步失败都到这里', err))
  .finally(() => console.log('无论成败都执行'));

// 4) async/await：同步式写法
async function loadData() {
  try {
    const res = await fetch('/api/user');   // 等待，但不阻塞主线程
    const user = await res.json();
    const postsRes = await fetch(`/api/posts?uid=${user.id}`);
    const posts = await postsRes.json();
    return { user, posts };
  } catch (err) {
    console.error('出错', err);
  }
}

// 5) 并发：不要串行 await
// ❌ 慢：两个请求串行
const a = await fetch('/a'); const b = await fetch('/b');
// ✅ 快：并发
const [aRes, bRes] = await Promise.all([fetch('/a'), fetch('/b')]);
// 全部完成才继续；任一失败则整体 reject
// 想要「谁先好谁先返回、容忍失败」用 Promise.allSettled
const results = await Promise.allSettled([p1, p2, p3]);
```

## ⚠️ 易错点 / 面试要点

- **事件循环输出顺序题**：牢记「同步 → 微任务 → 宏任务」，能秒杀 90% 的 `setTimeout` + `Promise` 排序题。
- **`await` 让出主线程**：它**不阻塞**整个程序，只暂停当前 async 函数，主线程继续跑别的。
- **串行 await 是性能陷阱**：多个**独立**异步请求别一个个 await，用 `Promise.all` 并发。
- **`Promise.all` 一损俱损**：一个 reject 全 reject；要容错用 `Promise.allSettled`（ES2020）。
- **忘记 try/catch**：`async/await` 的 rejected Promise 若不 catch，会变成未处理的 rejection。
- **`forEach` 不等 await**：`arr.forEach(async ...)` 不会等待每个完成。要串行用 `for...of`，要并发用 `Promise.all(arr.map(...))`。
- **`return` vs `return await`**：在 try/catch 内，`return await` 才能被本层 catch 捕获；直接 `return promise` 的错误会「穿透」。
