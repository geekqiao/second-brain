# 📚 参考资料（Reference）

> [!info] 只读区域 · [[Home]] · [[🗺️ 知识地图 MOC]]

这里存放**从互联网收集的原始资料**：博客文章、官方文档片段、教程、问答、论文笔记等。它是主知识库的「**原料库**」——**保持原样、只读**，不在这里编辑改写。

---

## 🧭 这个目录的定位

| | 主知识库（`10-`~`40-` 等） | 参考资料（本目录） |
| --- | --- | --- |
| 性质 | **我整理的笔记**（我的理解） | **别人的原文**（原始资料） |
| 是否编辑 | ✅ 持续改写、精炼 | ❌ 保持原样，只读 |
| 关系 | 整理后的「成品」 | 成品背后的「原料」 |

> **两者可以主题重叠**：比如主库里你有 [[异步编程]]，这里也可以同时存三篇讲 Promise 的好文章。主库是「我懂了之后写的」，这里是「我学的来源」。

---

## 🔒「只读」怎么落实

Obsidian **原生不支持按目录锁定只读**，我们用「**约定 + 标记**」来落实：

1. **目录约定**：放在 `80-参考资料/` 里的文件就是只读原始资料，不在这里编辑改写。
2. **frontmatter 标记**：每个文件标 `type: reference`，一眼可辨（也可用 Dataview 查询过滤）。
3. **想加工？** → 在主知识库里新建笔记（写你的理解），再链接回这里的原文。**不在原文上改**。

> [!tip] 想「硬性」防误改？
> 可选的强化方案：把本目录纳入 Git 版本控制（见 [[排错与最佳实践]]），改动需要 git 提交才能留存，物理上更难误改。需要时再启用即可。

## ➕ 如何添加新的参考资料

1. **复制模板**：复制 [[_参考资料模板]] 到本目录，重命名（如 `2026-07-15-mdn-promise`）。
2. **填 frontmatter**：`source`（原文 URL）、`author`、`captured`（抓取日期）、`tags`、`topic`。
3. **粘贴正文**：把网页内容粘到「正文」区，保留原貌。
4. **加关联**：在「关联笔记」里链到主知识库的相关笔记（如 `[[异步编程]]`）。

> 命名建议：`日期-来源-主题`（如 `2026-07-15-ruanyf-es6`），便于按时间排序与辨识。

## 🗂️ 内部组织（由你自定义）

本目录的**内部子目录由你自己决定**，不强加结构。几种常见方式，按需自建：

- **按主题**（与主库镜像）：`80-参考资料/python/`、`80-参考资料/docker/`、`80-参考资料/ml/`…
- **按来源**：`80-参考资料/mdn/`、`80-参考资料/知乎/`、`80-参考资料/bilibili/`…
- **按类型**：`80-参考资料/文章/`、`80-参考资料/视频/`、`80-参考资料/论文/`…
- **完全平铺**：所有文件一个目录，靠 `tags` 和 `topic` 关联。

> 不必一次定死，随着资料增多再自然分类也来得及。关键是**每个文件的 frontmatter 填全**，这样无论怎么分目录都能用 tag/Dataview 找到。
