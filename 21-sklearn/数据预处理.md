# 数据预处理

> [!info] 所属：[[sklearn MOC]] · 上一篇 [[sklearn 入门与 API 设计]] · 下一篇 [[监督学习算法]]

## 核心概念

真实数据很脏：尺度不一、有缺失、有类别文本。sklearn 的 `preprocessing` 模块把数据**变成模型能吃的样子**。预处理做得对，模型效果往往比换算法提升更大。

### 划分数据集
`train_test_split`：分出训练集（学）与测试集（评估泛化）。必要时还有验证集。
- 分层抽样 `stratify=y`：类别不平衡时保证各集比例一致。

### 特征缩放（Feature Scaling）
很多模型（SVM、KNN、线性/逻辑回归、神经网络）对尺度敏感，必须缩放；树模型（随机森林）一般不需要。
- `StandardScaler`：标准化为均值 0、方差 1（最常用）。
- `MinMaxScaler`：缩放到 [0, 1]。
- `RobustScaler`：用中位数/分位数，对**异常值**鲁棒。

### 类别特征编码
模型只懂数值，类别（如「北京/上海」）要编码：
- `OrdinalEncoder`：序数编码（整数），适合有大小关系的类别。
- `OneHotEncoder`：独热编码，适合无序类别（颜色、城市）。
- `LabelEncoder`：编码**标签 y**（不是特征 X）。

### 缺失值处理
- `SimpleImputer`：用均值/中位数/众数/常数填补。
- 或在 pandas 阶段 `dropna` / `fillna`。

> **关键纪律**：所有转换器都只**在训练集上 `fit`**，再用同一参数 `transform` 测试集——避免数据泄露（见 [[sklearn 入门与 API 设计]]）。

## 代码示例

```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer

# 构造含缺失、含类别、尺度不一的脏数据
df = pd.DataFrame({
    'age':   [25, np.nan, 35, 45, 50],
    'income':[30000, 50000, 70000, np.nan, 90000],
    'city':  ['BJ', 'SH', 'BJ', 'GZ', 'SH'],
    'y':     [0, 1, 1, 0, 1]
})

# 1) 划分（分层，保比例）
X_train, X_test, y_train, y_test = train_test_split(
    df[['age','income','city']], df['y'], test_size=0.4, random_state=42, stratify=df['y'])

# 2) 数值列：缺失值填补 + 标准化（仅训练集 fit！）
num_cols = ['age', 'income']
imp = SimpleImputer(strategy='median')
scaler = StandardScaler()

Xtr_num = scaler.fit_transform(imp.fit_transform(X_train[num_cols]))
Xte_num = scaler.transform(imp.transform(X_test[num_cols]))   # 用训练集参数

# 3) 类别列：独热编码
ohe = OneHotEncoder(handle_unknown='ignore', sparse_output=False)
Xtr_cat = ohe.fit_transform(X_train[['city']])
Xte_cat = ohe.transform(X_test[['city']])

# 4) 拼回（实际中用 ColumnTransformer + Pipeline 自动化，见 [[无监督学习与 Pipeline]]）
import numpy as np
Xtr = np.hstack([Xtr_num, Xtr_cat])
Xte = np.hstack([Xte_num, Xte_cat])
```

## ⚠️ 易错点 / 面试要点

- **数据泄露是头号陷阱**：用全量数据 `fit` 缩放/填补，再划分 → 测试集信息污染训练，线上表现远差于离线评估。务必**先划分、再 fit**，或用 `Pipeline`。
- **缩放只对训练集 fit**：测试集/线上数据用训练集的 `mean_`/`scale_` 来 transform，不能自己 fit（否则特征空间不一致）。
- **独热 vs 序数选错**：无序类别（城市、颜色）用独热；硬给整数（BJ=0,SH=1,GZ=2）会引入虚假大小关系，误导线性模型。树模型对这点不敏感。
- **`OneHotEncoder` 的未知类别**：测试集出现训练集没有的新城市会报错，设 `handle_unknown='ignore'`。
- **树模型通常不需要缩放**：决策树按阈值分裂，与尺度无关。但缩放也不会有害。
- **缺失值策略**：中位数对异常值鲁棒（数值列常用）；众数用于类别列。直接 dropna 可能丢大量样本。
- **`ColumnTransformer`** 处理「数值列走一套流程、类别列走另一套」更优雅，配合 Pipeline 是生产标准（见 [[无监督学习与 Pipeline]]）。
