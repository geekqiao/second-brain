# 文件系统与权限

> [!info] 所属：[[Linux MOC]] · 下一篇 [[常用命令与文件操作]]

## 核心概念

Linux 「**一切皆文件**」——硬件是文件（`/dev`），进程信息是文件（`/proc`），网络配置也是文件。理解**目录层级**和**权限模型**，是用好 Linux 的地基。

### 目录结构（FHS 标准，关键目录）
| 路径 | 内容 |
| --- | --- |
| `/` | 根目录（一切起点） |
| `/home` | 普通用户家目录（`/home/geek`，简写 `~`） |
| `/root` | root 用户家目录 |
| `/etc` | 系统与软件的**配置文件** |
| `/var` | 经常变化的数据（日志 `/var/log`、缓存） |
| `/usr` | 用户程序与库（`/usr/bin`、`/usr/local`） |
| `/bin`, `/sbin` | 基础可执行命令（现已多软链到 `/usr/bin`） |
| `/tmp` | 临时文件（重启可能清空） |
| `/dev` | 设备文件 |
| `/proc` | 内核与进程信息（虚拟文件系统） |

### 绝对路径 vs 相对路径
- **绝对**：从 `/` 开始，如 `/home/geek/file.txt`。
- **相对**：从当前目录开始，`./file.txt`（当前）、`../`（上一级）、`~/`（家目录）。

### 权限模型（rwx）
每个文件有三组权限，分别对应**所有者 u / 所属组 g / 其他人 o**：
- `r`（read，4）：读。
- `w`（write，2）：写。
- `x`（execute，1）：执行（对目录而言是「能否进入」）。

`ls -l` 显示如 `rwxr-xr--` = 所有者 rwx(7)、组 r-x(5)、其他人 r--(4)。

### 修改权限与所有者
- `chmod` 改权限：`chmod 755 file`（数字）或 `chmod u+x file`（符号）。
- `chown` 改所有者：`chown user:group file`。

## 代码示例（命令）

```bash
# 目录导航
pwd                  # 打印当前绝对路径
ls -la               # 列出全部（含隐藏文件 -a，长格式 -l）
cd /etc              # 切到 /etc
cd ~                 # 回家目录
cd -                 # 回到上一次所在目录

# 权限解读：ls -l 输出
# -rwxr-xr--  1 geek staff  1024 Jul 15 note.txt
#  ↑└─┬─┘└─┬─┘└─┬─┘
#  类型 所有者 组  其他人

# 数字法改权限（推荐，快）
chmod 755 script.sh     # rwxr-xr-x（脚本常用：自己可写，他人可执行）
chmod 644 note.txt      # rw-r--r--（普通文件常用）
chmod 700 secret/       # rwx------（仅自己可访问的目录）

# 符号法（更直观）
chmod u+x script.sh     # 给所有者加执行权
chmod g-w file          # 去掉组的写权
chmod a+r file          # a=all，所有人可读

# 改所有者与组
sudo chown alice:devs file   # 所有者→alice，组→devs

# 目录的 x 权限很特殊
chmod 711 /home/geek    # 别人能 cd 进来（x）但看不到内容（无 r）
```

## ⚠️ 易错点 / 面试要点

- **目录的 `r` 与 `x` 区别**：`r` 能 `ls` 列出名字；`x` 才能 `cd` 进入并访问内部文件。给目录权限常需配合（如 `755`）。只有 `r` 没有 `x`，能列出文件名但打不开。
- **`rm` 没有「回收站」**：删了就是删了（尤其 `rm -rf`，极度危险）。养成 `-i` 确认或先 `ls` 看一眼的习惯；绝不轻易 `rm -rf /`。
- **绝对路径 vs 相对路径**：脚本里尽量用**绝对路径**，避免「当前目录」不确定导致找不到文件。
- **`.` 开头的文件是隐藏文件**：`ls` 默认不显示，要 `ls -a`。配置文件多为隐藏（`.bashrc`）。
- **`chmod 777` 是坏味道**：给所有人完全权限，安全大忌。只在临时调试用，且收窄范围。生产环境遵循「最小权限」。
- **`sudo` 谨慎**：以 root 身份执行，威力大风险大。能不用就不用；配置文件改动、安装系统级软件才用。
- **大小写敏感**：`File.txt` 与 `file.txt` 是两个文件（与 Windows/macOS 默认不同）。
