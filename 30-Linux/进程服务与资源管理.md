# 进程、服务与资源管理

> [!info] 所属：[[Linux MOC]] · 上一篇 [[常用命令与文件操作]] · 下一篇 [[Shell 脚本基础]]

## 核心概念

程序运行起来就是**进程（process）**。Linux 提供一整套工具来查看、控制进程，以及把长期运行的服务**托管**起来（开机自启、崩溃重启）。这是部署后端服务的基础。

### 查看进程
- `ps`：进程快照。`ps aux` 看全部（`a` 所有用户、`u` 详细、`x` 无终端）。
- `top` / `htop`：实时动态监控（CPU、内存、进程）。
- `pgrep` / `pidof`：按名查 PID。

### 控制进程
- `kill <PID>`：发信号（默认 TERM，优雅终止）。
- `kill -9 <PID>`：SIGKILL，**强杀**（进程无法捕获，最后手段）。
- `Ctrl+C`：前台中断；`Ctrl+Z`：挂起到后台。
- `jobs` / `fg` / `bg`：管理后台作业。

### 后台与持久运行
- 命令后加 `&`：放后台运行（但关终端会死）。
- `nohup cmd &`：忽略挂断信号，关终端仍活。
- `disown`：剥离已运行进程与终端的关系。
- **`tmux` / `screen`**：终端复用器，断开 SSH 后进程继续跑，重连可恢复（比 nohup 更好用）。

### systemd（现代服务管理）
Linux 主流的服务管理器。把程序写成「**单元（unit）**」，由 systemd 负责启动、重启、开机自启、日志收集。
- `systemctl start/stop/restart/enable/status <服务>`。
- `journalctl` 查看服务日志。
- 配置文件在 `/etc/systemd/system/xxx.service`。

### 资源监控
- `free -h`：内存。
- `df -h`：磁盘使用。
- `du -sh <dir>`：目录大小。
- `uptime`：负载均值。
- `ss` / `netstat`：网络连接与端口（见 [[网络磁盘与用户]]）。

## 代码示例（命令）

```bash
# 查看进程
ps aux                       # 全部进程快照
ps aux | grep python         # 找 python 进程
top                          # 实时监控（按 q 退出）；htop 更友好
pgrep -f "python app.py"     # 按命令行查 PID

# 终止进程
kill 12345                   # 发 TERM（优雅退出）
kill -9 12345                # 强杀（SIGKILL，无法拒绝）
killall python               # 按名杀所有匹配进程
pkill -f "app.py"            # 按命令行模式杀

# 后台运行
python app.py &              # 后台启动（关终端会死）
nohup python app.py > app.log 2>&1 &   # 关终端仍活，日志写文件
# 更推荐 tmux：断开 SSH 后进程不挂
tmux new -s work             # 新建会话
# ... 运行东西，然后 Ctrl+B D 脱离
tmux attach -t work          # 重连回会话

# systemd 服务管理（需 sudo）
sudo systemctl status nginx          # 查看状态
sudo systemctl restart nginx         # 重启
sudo systemctl enable nginx          # 设置开机自启
sudo systemctl disable nginx         # 取消自启
sudo journalctl -u nginx -f          # 实时看 nginx 日志

# 资源监控
free -h                      # 内存（人类可读）
df -h                        # 各挂载点磁盘使用
du -sh /var/log              # 某目录总大小
uptime                       # 运行时间 + 负载（1/5/15分钟平均负载）
ss -tulpn                    # 监听中的 TCP/UDP 端口与对应进程
```

## ⚠️ 易错点 / 面试要点

- **`kill -9` 是最后手段**：它不给进程清理机会（不触发 finally、不刷缓冲），可能导致数据损坏。先用 `kill`（TERM），不行再 `-9`（KILL）。
- **`&` 不等于「持久」**：`cmd &` 关闭终端进程会被 SIGHUP 杀死。要持久用 `nohup` 或 `tmux`，要专业托管用 systemd。
- **生产服务用 systemd 托管**：手动 `nohup` 跑服务无法自动重启、无统一日志。systemd 提供 `Restart=always`、开机自启、`journalctl` 日志，是标准做法。
- **`enable` vs `start`**：`start` 是现在启动；`enable` 是设为开机自启。两个维度，常组合 `enable --now`。
- **僵尸/孤儿进程**：子进程结束但父进程未回收 → 僵尸（zombie）；父进程先死 → 孤儿被 init 收养。面试常问。
- **负载均值（load average）**：三个数是 1/5/15 分钟的平均运行队列长度。通常不应长期超过 CPU 核数。
- **`htop` 比 `top` 好用**：彩色、可交互（F5 树状、F6 排序、F9 发信号），能装就装。
