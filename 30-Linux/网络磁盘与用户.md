# 网络、磁盘与用户

> [!info] 所属：[[Linux MOC]] · 上一篇 [[文本处理三剑客]]

## 核心概念

把 Linux 当服务器用，绕不开三件事：**联网**（排障、连通性）、**磁盘**（空间、挂载）、**用户**（账号、权限、远程登录）。这是从「会用命令」到「能管服务器」的分水岭。

### 网络
- **连通性诊断**：`ping`（通不通）、`traceroute`（卡在哪一跳）。
- **端口与连接**：`ss`（现代，替代 netstat）看监听端口与连接；`-tulpn` 是常用组合。
- **HTTP 调试**：`curl`（发请求、看响应头/状态码）、`wget`（下载）。
- **接口与地址**：`ip addr`（看网卡 IP，替代旧 `ifconfig`）。
- **DNS**：`dig` / `nslookup` 查域名解析。

### SSH（远程登录）
- `ssh user@host`：安全远程登录。
- **密钥认证**优于密码：`ssh-keygen` 生成密钥对，`ssh-copy-id` 装公钥到服务器，之后免密登录。
- `scp` / `rsync`：远程复制文件（rsync 增量、更高效）。
- 配置 `~/.ssh/config` 可给主机起别名。

### 磁盘
- `df -h`：各挂载点**空间**使用。
- `du -sh <dir>`：某目录**占用**大小。
- `mount` / `umount`：挂载/卸载设备（云盘、U 盘）。
- `lsblk`：看块设备树。

### 用户与权限
- `whoami` / `id`：当前用户身份。
- `useradd` / `passwd` / `userdel`：增/改密码/删用户。
- `usermod -aG group user`：把用户加入组（如 docker 组）。
- `sudo`：临时提权执行（配置在 `/etc/sudoers`，用 `visudo` 编辑）。

## 代码示例（命令）

```bash
# ============ 网络 ============
ping -c 4 example.com          # 发 4 个包测连通性（Ctrl+C 停）
traceroute example.com         # 追踪路径，看卡在哪一跳
ip addr                        # 查看网卡与 IP 地址
ss -tulpn                      # 监听中的 TCP(-t)/UDP(-u) 端口+进程(-p) 不解析名(-n)
ss -tn                         # 当前 TCP 连接
curl -v https://example.com    # 详细请求/响应（排障 API 必备）
curl -X POST -H "Content-Type: application/json" \
     -d '{"k":"v"}' https://api.example.com    # 发 POST + JSON
wget https://example.com/file.zip             # 下载文件
dig example.com                # DNS 解析（A 记录）

# ============ SSH 远程登录与传输 ============
ssh user@192.168.1.10          # 远程登录
ssh -p 2222 user@host          # 指定端口
ssh-keygen -t ed25519          # 生成密钥对（推荐 ed25519）
ssh-copy-id user@host          # 把公钥装到服务器，之后免密
scp file.txt user@host:/tmp/   # 复制本地文件到远程
rsync -avz src/ user@host:dest/  # 增量同步目录（-a 归档 -v 详细 -z 压缩）

# ~/./.ssh/config 别名（省去每次输长命令）
# Host myserver
#   HostName 192.168.1.10
#   User geek
#   IdentityFile ~/.ssh/id_ed25519
# 之后: ssh myserver

# ============ 磁盘 ============
df -h                          # 各挂载点总/已用/可用空间
du -sh /var/log                # 日志目录多大
du -ah . | sort -rh | head     # 当前目录最大的文件
lsblk                          # 块设备树（看有几块盘）
mount /dev/sdb1 /mnt/data      # 挂载设备到目录（需 sudo）

# ============ 用户管理（多需 sudo）============
whoami                         # 当前用户名
id                             # 当前 uid/gid/所属组
sudo useradd -m alice          # 建用户并建家目录(-m)
sudo passwd alice              # 设/改密码
sudo usermod -aG docker alice  # 把 alice 加入 docker 组（重新登录生效）
sudo userdel -r alice          # 删用户并删家目录(-r)
sudo visudo                    # 安全编辑 sudo 权限配置
```

## ⚠️ 易错点 / 面试要点

- **用 `ss` 替代 `netstat`**：netstat 已废弃，ss 更快更全。但很多老教程/面试仍提 netstat，知道等价即可。
- **端口排障三步走**：`ping`（通不通）→ `ss -tulpn`（服务在不在监听）→ `curl -v`（应用层通不通）。能定位 90% 的「连不上」问题。
- **防火墙可能挡端口**：服务在监听但仍连不上，查 `ufw`/`firewalld`/云厂商安全组。
- **SSH 用密钥、关密码**：密钥更安全且免密；生产服务器建议禁用密码登录（`PasswordAuthentication no`）。
- **`scp` 每次全量复制，`rsync` 增量**：大目录同步/备份用 `rsync -avz --delete`（`--delete` 同步删除，慎用）。
- **加组后要重新登录**：`usermod -aG` 后当前会话不会立即生效，需退出重进或 `newgrp`。
- **`sudo` 权限编辑用 `visudo`**：直接改 `/etc/sudoers` 写错会导致无法 sudo，`visudo` 会校验语法。
- **磁盘满了的诡异现象**：服务写不了日志、命令报错、数据库挂——先 `df -h` 看是不是 100% 满了，`du` 找大文件清理。
