# 文本处理三剑客

> [!info] 所属：[[Linux MOC]] · 上一篇 [[Shell 脚本基础]] · 下一篇 [[网络磁盘与用户]]

## 核心概念

**grep / sed / awk** 被称为 Linux 文本处理「三剑客」，是处理日志、数据、配置文件的利器。Python 虽强，但面对「快速搜一行、改一处、统计一列」这种小任务，三剑客在命令行上更快、无需写脚本。

| 工具 | 擅长 | 一句话 |
| --- | --- | --- |
| **grep** | 搜索/过滤 | 「找含某模式的行」 |
| **sed** | 流编辑（替换/删除） | 「批量改文本」 |
| **awk** | 按列处理 + 编程 | 「按列统计/格式化」 |

### grep（Global RegExp Print）
- 在文件或输入中**搜索匹配模式的行**。
- 常用：`-i`（忽略大小写）、`-n`（显示行号）、`-r`（递归目录）、`-v`（反向，不匹配的行）、`-E`（扩展正则，支持 `+`/`|`）、`-c`（计数）。

### sed（Stream Editor）
- 逐行处理文本，**默认输出到屏幕不改原文件**。
- 替换：`s/旧/新/`（首个）/ `s/旧/新/g`（全部）。
- `-i`：直接改原文件（**先备份再试**）。
- 还能删除行 `d`、插入 `i`、追加 `a`。

### awk
- 把每行按分隔符拆成**字段**（`$1`/`$2`…，`$0` 是整行），可写条件与计算。
- `-F` 指定分隔符（默认空白）。
- 适合「第几列满足条件就怎样」「求某列总和」这类任务。

## 代码示例（命令）

```bash
# ============ grep：搜索过滤 ============
grep "error" app.log                # 找含 error 的行
grep -i "error" app.log             # 忽略大小写
grep -n "error" app.log             # 带行号
grep -c "error" app.log             # 只数匹配行数
grep -v "debug" app.log             # 反向：不含 debug 的行
grep -rn "TODO" src/                # 递归搜 src 目录（找代码 TODO）
grep -E "error|warn" app.log        # 扩展正则：error 或 warn
ps aux | grep python                # 配合管道找进程

# ============ sed：替换编辑 ============
sed 's/foo/bar/' file               # 每行首个 foo→bar（仅显示）
sed 's/foo/bar/g' file              # 每行所有 foo→bar（g=global）
sed -i 's/foo/bar/g' file           # 直接改原文件（谨慎！）
sed -i.bak 's/foo/bar/g' file       # 改原文件并备份原文件为 file.bak
sed '/^$/d' file                    # 删除空行
sed '5d' file                       # 删除第 5 行
sed -n '10,20p' file                # 只打印第 10~20 行

# ============ awk：按列处理 ============
echo "Alice 90" | awk '{print $1}'          # Alice（第1列）
echo "Alice 90" | awk '{print $1, $2*2}'    # Alice 180（可计算）
awk -F: '{print $1}' /etc/passwd            # 以:分隔，打印用户名列
awk -F, '{sum += $3} END {print sum}' data.csv   # 第3列求和
awk '$2 > 80 {print $1}' scores.txt         # 第2列>80 时打印第1列（条件）
df -h | awk '{print $5}'                    # 取磁盘使用率列

# ============ 经典组合（管道串联三剑客）============
# 统计日志里出现最多的 IP（access.log 第1列）
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -n 10
#   awk 取列 → sort 排序 → uniq -c 去重计数 → sort -rn 按数降序 → head 取前10

# 找出代码里所有 console.log 并显示文件名行号
grep -rn "console.log" src/ | sed 's/:.*//'
```

## ⚠️ 易错点 / 面试要点

- **`sed -i` 直接改原文件、无回收站**：先用不带 `-i` 的版本预览结果，确认后再 `-i`；或用 `-i.bak` 自动备份。误操作可能毁掉配置文件。
- **`grep` 默认基础正则**：`+`/`?`/`|` 要转义或加 `-E`。记不住就统一用 `grep -E`（或 `egrep`）。
- **awk 字段从 `$1` 开始**：`$0` 是整行，`$1` 是第一列。和多数编程语言（从 0 开始）相反。
- **awk 分隔符**：默认连续空白为一个分隔符；处理 CSV 用 `-F,`；TSV 用 `-F'\t'`。
- **管道里 `sort | uniq -c` 是去重计数标配**：uniq 只对**相邻**重复行去重，所以必须先 sort。
- **复杂逻辑别硬写 awk/sed**：超过几行的处理，用 [[Python MOC]] 更可读可维护。三剑客适合一行搞定的小任务。
- **`sed` 的分隔符可换**：替换含 `/` 的路径时，`s/旧/新/` 很乱，可换成 `s|旧|新|` 或 `s#旧#新#`。
