# 常用命令与文件操作

> [!info] 所属：[[Linux MOC]] · 上一篇 [[文件系统与权限]] · 下一篇 [[进程服务与资源管理]]

## 核心概念

日常 Linux 操作 80% 是「**找文件、看内容、搬文件**」。掌握这些命令 + **管道（pipe）** 思想，就能组合出强大的处理能力。命令行哲学：**每个工具做好一件小事，用管道串起来**。

### 管道与重定向
- `|`（管道）：把前一个命令的**输出**作为后一个命令的**输入**。
- `>` 重定向输出（覆盖）：`echo hi > f.txt`。
- `>>` 追加：`echo more >> f.txt`。
- `<` 从文件读输入。
- `2>` 重定向错误流；`2>&1` 合并错误到标准输出。
- 三个数据流：**stdin（0）标准输入 / stdout（1）标准输出 / stderr（2）标准错误**。

### 增删改查
- 建：`mkdir`（目录）、`touch`（空文件）。
- 复制/移动/删除：`cp` / `mv` / `rm`。
- `-r` 递归（目录操作必备）。

### 查看内容
- `cat`：打印全部（适合小文件）。
- `less`：分页浏览（大文件，`/` 搜索，`q` 退出）。
- `head` / `tail`：看头部/尾部；`tail -f` 实时跟踪（看日志神器）。

### 查找
- `find`：按名/类型/时间找文件（功能强大）。
- `grep`：在文件内容里搜文本（见 [[文本处理三剑客]]）。

## 代码示例（命令）

```bash
# 文件增删改
mkdir -p a/b/c              # -p 递归建多级目录
touch app.py                # 创建空文件 / 更新时间戳
cp -r src dest              # -r 递归复制目录
mv old.txt new.txt          # 移动 / 重命名
rm file.txt                 # 删除（无回收站！）
rm -rf build/               # 强制递归删目录（危险，确认路径）

# 查看内容
cat note.txt                # 全部输出
less big.log                # 分页（空格翻页，/搜索，q退出）
head -n 20 file             # 前 20 行
tail -n 50 file             # 后 50 行
tail -f /var/log/app.log    # 实时跟踪新内容（看日志必备）

# 查找文件
find . -name "*.py"                 # 按名找 .py 文件
find . -type f -size +10M           # 找大于 10MB 的普通文件
find . -name "*.log" -mtime -7      # 7 天内修改过的 .log
find . -type d -name node_modules -prune -exec rm -rf {} +  # 找并删除（谨慎）

# 管道 + 重定向（核心思想）
ls -l | grep ".py"          # 列出文件 → 过滤含 .py 的行
cat access.log | wc -l      # 统计行数（文件总行数）
ps aux | grep python        # 找 python 进程
command > out.log 2>&1      # 标准输出和错误都写到 out.log
echo "done" >> tasks.log    # 追加一行

# 组合实战：找出最大的 5 个文件
du -ah . | sort -rh | head -n 5

# 快捷查看帮助
man grep                    # 完整手册（q 退出）
grep --help                 # 简要帮助
tldr grep                   # 社区示例（需安装 tldr）
```

## ⚠️ 易错点 / 面试要点

- **`rm -rf` 是头号危险命令**：路径写错（如 `rm -rf /` 或变量为空时 `rm -rf $EMPTY/*`）能删光系统。删之前先 `echo` 确认路径，或用 `rm -ri` 交互确认。
- **`cp`/`rm` 对目录要加 `-r`**：忘了会报「is a directory」。
- **管道只传 stdout**：错误信息（stderr）默认不进管道。要捕获错误用 `2>&1` 先合并。
- **`>` 会覆盖文件**：误用会清空重要文件。不确定就用 `>>` 追加，或先备份。
- **`cat` 大文件刷屏**：大文件用 `less` / `head`，别 `cat` 几十万行日志。
- **`find` 的 `-exec` vs `xargs`**：`-exec` 对每个结果执行一次命令；`xargs` 把结果批量传参，处理大量文件时更高效。
- **`tail -f` 看日志**：实时跟踪，调试运行中的服务最常用（Ctrl+C 退出）。
- **善用通配符**：`*.py`（所有 py）、`file?.txt`（单字符）、`[abc]`（字符集）。
