# 广播与向量化运算

> [!info] 所属：[[numpy MOC]] · 上一篇 [[索引与切片]] · 下一篇 [[数组变形与拼接]]

## 核心概念

numpy 的两大超能力：**向量化**（对整个数组做运算，不写 for 循环）和**广播**（让形状不同的数组也能一起运算）。它们是 numpy 比 Python list 快几十倍的根本原因。

### 向量化（Vectorization）
对数组直接做运算，numpy 在底层用 C 批量执行，**不逐元素循环**：
- `a + 1`、`a * 2`、`a ** 2`、`np.sqrt(a)`——每个元素都运算。
- `a + b`——两个数组逐元素相加（形状需兼容或可广播）。
- 对比 Python list：`[x*2 for x in lst]` 要循环；`a*2` 一行、快几十倍。

> 心法：**能用向量化就别写 for 循环**。看到 `for i in range(len(a))` 操作 numpy，多半有更优写法。

### 广播（Broadcasting）
当两个数组形状**不同但兼容**时，numpy 自动「扩展」较小的数组，让运算得以进行，**无需真正复制数据**。
- 标量与数组：`a + 5`（5 广播到 a 的形状）。
- 一维与二维：`(3,) + (2,3)` → 一维沿第 0 轴广播成 `(2,3)`。

### 广播规则（从末尾逐轴比较）
两数组各轴满足下列之一即可广播：
1. **相等**，或
2. **其中一方为 1**。
- 维度不同时，**前面补 1**（视为 1）。
- 不满足 → `ValueError: operands could not be broadcast together`。

### 广播直觉
把数组想象成网格，广播 = 沿「大小为 1」的轴复制扩展，直到形状一致。

## 代码示例

```python
import numpy as np

# 1) 向量化：告别 for 循环
a = np.array([1, 2, 3, 4])
print(a * 2)            # [2 4 6 8]（每个元素 ×2，C 层批量）
print(a ** 2)           # [1 4 9 16]
print(np.exp(a))        # e^x 逐元素

# 对比纯 Python（慢几十倍）
# py = [x*2 for x in a.tolist()]

# 2) 广播：标量
a + 100                 # [101 102 103 104]

# 3) 广播：1D + 2D
row = np.array([1, 2, 3])           # shape (3,)
mat = np.array([[10, 20, 30],       # shape (2, 3)
                [40, 50, 60]])
print(mat + row)                    # row 广播成 (2,3) 逐行相加
# [[11 22 33]
#  [41 52 63]]

# 4) 广播：列向量 + 行向量 → 矩阵（经典技巧）
col = np.array([[0], [10], [20]])   # shape (3,1)
row = np.array([0, 1, 2])           # shape (3,)
print(col + row)                    # (3,1)+(3,)→(3,3) 外加效果
# [[ 0  1  2]
#  [10 11 12]
#  [20 21 22]]

# 5) 广播规则检验
# (2,3) + (3,)   → (2,3)+(1,3)→(2,3) ✓
# (2,3) + (2,)   → 末轴 3≠2 且非1 → ✗ 报错
try:
    np.ones((2,3)) + np.ones((2,))
except ValueError as e:
    print('广播失败:', e)

# 6) 实战：标准化每列（减均值除标准差）
data = np.random.randn(100, 3)
mean = data.mean(axis=0)            # shape (3,)
std  = data.std(axis=0)             # shape (3,)
normalized = (data - mean) / std    # 广播：逐列标准化
```

## ⚠️ 易错点 / 面试要点

- **向量化优先于循环**：numpy 慢的常见原因就是写了 Python 层 for 循环。能用数组运算/`where`/聚合替代就替代。
- **广播形状要对得上**：`(3,)` 能和 `(2,3)` 广播（末轴 3 相等）；但 `(2,)` 不能和 `(2,3)` 广播（末轴 2≠3）。报错时先 `print(a.shape, b.shape)` 对照规则。
- **列向量 shape 是 `(n,1)` 不是 `(n,)`**：`a.reshape(-1,1)` 把一维变列向量，才能和行向量 `(m,)` 广播成 `(n,m)`。这是做矩阵运算的高频操作。
- **广播不复制数据**：它是「虚拟扩展」，省内存；但 `np.broadcast_to` 真要复制时占内存。
- **小心无意广播**：两个形状本该相同却因广播「悄悄对齐」，掩盖了逻辑错误。写 `(a==b).all()` 验证形状预期。
- **`axis` 方向**：聚合时 `axis=0` 沿行方向（消去行，得每列结果）；`axis=1` 沿列方向（得每行结果）。和「跨哪个轴」一致。
- **性能**：向量化 + 连续内存 + 固定类型 = numpy 快的本质。预分配大数组、避免循环内频繁创建临时数组。
