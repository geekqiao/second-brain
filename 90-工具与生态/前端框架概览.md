# 前端框架概览

> [!info] 广度概览 · 回到 [[🗺️ 知识地图 MOC]]

## 是什么

前端框架（Frontend Framework）是用 [[JavaScript MOC]]、[[HTML MOC]]、[[CSS MOC]] 搭起来的**开发脚手架**——它解决「用原生 JS 写大型应用太痛苦」的问题，提供组件化、状态管理、高效更新、路由等能力。现代前端开发几乎都基于框架。

## 为什么重要

- 组件化让 UI **可复用、可维护**，告别复制粘贴 HTML。
- 声明式写法（描述「要什么」而非「怎么做」）比命令式操作 DOM 直观得多。
- 生态（路由、状态、UI 库、SSR）成熟，能撑起复杂单页应用（SPA）。

## 核心要点（点到为止，选一个深入即可）

### 四大主流框架
| 框架 | 特点 | 适合 |
| --- | --- | --- |
| **React** | 组件 + JSX、虚拟 DOM、生态最大、灵活 | 大型应用、求职（市场最大） |
| **Vue** | 模板语法友好、渐进式、中文社区强 | 快速上手、中小项目 |
| **Angular** | 大而全（自带 DI/路由/表单）、TypeScript 优先 | 企业级重型项目 |
| **Svelte** | 编译时框架、无虚拟 DOM、代码量少 | 追求性能与简洁 |

> React 和 Vue 占据绝大部分市场。**新手二选一**：求职优先 React，快速出活选 Vue。

### 关键概念（框架通用）
- **组件（Component）**：UI 的独立可复用单元。
- **状态（State）**：组件内部数据；变化时 UI 自动更新（响应式）。
- **props**：父传子的数据。
- **虚拟 DOM / 编译优化**：框架算出最小变更再更新真实 DOM，提升性能。
- **路由（Router）**：单页应用里切换「页面」不刷新。
- **状态管理**：跨组件共享数据（React 的 Redux/Zustand、Vue 的 Pinia）。

### 相关生态
- **构建工具**：Vite（现代首选）、Webpack（见 [[模块化与包管理]]）。
- **元框架（SSR/SSG）**：Next.js（基于 React）、Nuxt（基于 Vue）——解决 SEO、首屏速度。
- **UI 组件库**：Ant Design、Material UI、Element Plus。
- **类型系统**：TypeScript（JS + 类型，大型项目标配）。

> [!tip] 学习路径
> 先扎实 [[JavaScript MOC]]（尤其 ES6+、异步、模块化），再学框架。直接上框架会「知其然不知其所以然」。

## 去哪深入

- 官方文档是最佳起点：React（react.dev）、Vue（vuejs.org）。
- 选定一个后，做一个完整项目（待办/博客/商城）跑通组件、路由、状态、API 调用。
- 进阶：TypeScript + 元框架（Next.js / Nuxt）。
