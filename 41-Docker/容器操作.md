# 容器操作

> [!info] 所属：[[Docker MOC]] · 上一篇 [[镜像与 Dockerfile]] · 下一篇 [[数据卷与网络]]

## 核心概念

构建好镜像后，日常 90% 的工作是**运行、调试、管理容器**。掌握 `run`/`exec`/`logs` 这几条命令，就能驾驭容器生命周期。

### 容器生命周期
```
created → running → stopped → (deleted)
   ↑ docker run     ↑ docker stop/rm
```
- `docker run`：从镜像创建并启动容器（= create + start）。
- `docker start`/`stop`/`restart`：启动/停止/重启**已存在**的容器（不重建）。
- `docker rm`：删除已停止的容器（数据随之消失）。

### `docker run` 核心参数（最高频）
| 参数 | 作用 |
| --- | --- |
| `-d` | 后台运行（detached） |
| `--name` | 起名字 |
| `-p 宿主:容器` | 端口映射 |
| `-v 宿主:容器` | 挂载数据卷/目录（持久化） |
| `-e KEY=VAL` | 环境变量 |
| `-it` | 交互式终端（进容器用） |
| `--rm` | 容器停止后自动删除（临时容器） |
| `--network` | 加入指定网络 |

### 进容器调试：`exec`
`docker exec -it <容器> bash`：在**运行中**的容器里开一个交互终端，排查问题。

### 查日志：`logs`
`docker logs -f <容器>`：实时跟踪容器 stdout/stderr（应用日志要打到 stdout 才能看到）。

### 清理
- `docker rm <容器>`：删容器。
- `docker rmi <镜像>`：删镜像。
- `docker system prune`：一键清理悬空镜像、停止容器、无用网络（回收磁盘）。

## 代码示例（命令）

```bash
# 1) 启动一个 Web 容器（最常见组合）
docker run -d \
  --name web \
  -p 8080:80 \
  -e ENV=production \
  nginx:1.25
# 访问 http://localhost:8080 即可

# 2) 交互式：进运行中的容器
docker exec -it web bash          # 开 bash（alpine 用 sh）
ls /                              # 在容器内查看文件系统
exit                              # 退出（容器继续跑）

# 3) 看日志（排障必备）
docker logs web                   # 全部日志
docker logs -f web                # 实时跟踪（类似 tail -f）
docker logs --tail 100 web        # 最后 100 行

# 4) 生命周期管理
docker ps                         # 运行中的容器
docker ps -a                      # 全部（含已停）
docker stop web                   # 优雅停止
docker start web                  # 重新启动（不重建，配置/数据保留）
docker restart web
docker rm web                     # 删除（必须先 stop）
docker rm -f web                  # 强制删（运行中也删）

# 5) 一次性临时容器（跑完即删）
docker run --rm alpine echo "hello"

# 6) 资源查看
docker stats                      # 实时 CPU/内存（类似 top）
docker top web                    # 容器内进程
docker inspect web                # 容器详细配置（JSON）

# 7) 清理磁盘
docker system prune               # 清悬空资源
docker system prune -a            # 连未被使用的镜像一起清（更彻底）
docker image prune -a             # 只清镜像

# 8) 容器与宿主文件互拷
docker cp web:/etc/nginx/nginx.conf ./nginx.conf   # 容器→宿主
docker cp ./app.conf web:/etc/app/app.conf         # 宿主→容器
```

## ⚠️ 易错点 / 面试要点

- **端口冲突**：`-p 8080:80` 时若宿主 8080 已占用会启动失败。换宿主端口，或先停掉占用进程。
- **容器停了配置还在**：`stop` + `start` 是重启同一容器（保留 `-p`/`-v`/`-e` 等所有运行参数）；`rm` 后再 `run` 才是全新容器。别混淆。
- **删容器 ≠ 删数据卷**：`docker rm web` 删容器，但**挂载的命名 volume 不会被删**（防误删数据）。要连同删 volume 加 `-v`。
- **日志看不到？** 应用必须把日志输出到 **stdout/stderr**，`docker logs` 才能看。写本地文件日志的话 logs 看不到（反模式）。
- **`exec` 进容器调试**：容器要支持 `bash`（alpine 只有 `sh`，精简镜像可能都没有 shell）。生产容器应尽量精简、无 shell（安全）。
- **容器主进程退出 = 容器退出**：容器生命周期绑主进程（PID 1）。如 nginx 要前台跑（`daemon off`），后台模式会让容器立即退出。
- **`--rm` 谨慎**：停止即删，适合一次性任务（测试、迁移），别用于需要保留状态的容器。
