# 核心概念

> [!info] 所属：[[Docker MOC]] · 下一篇 [[镜像与 Dockerfile]]

## 核心概念

Docker 解决一个根本问题：**同一个应用，在不同环境表现不一致**。它的方案是容器化——把应用和依赖打包成标准单元，到处运行。理解 Docker，先分清四对核心概念。

### 镜像（Image）vs 容器（Container）
| | 镜像 Image | 容器 Container |
| --- | --- | --- |
| 类比 | 程序 / 类 / 模板 | 进程 / 实例 / 对象 |
| 性质 | 只读、静态 | 可写、运行中 |
| 关系 | 一个镜像可跑出多个容器 | 容器由镜像实例化 |

> 口诀：**镜像是蓝图，容器是按蓝图盖起来的房子**。`docker run 镜像` = 盖房。

### Dockerfile vs 镜像
- **Dockerfile**：一份文本「配方」，描述怎么从基础镜像构建出新镜像。
- **镜像**：按 Dockerfile 构建出的成品（`docker build`）。
- 类比：Dockerfile 是菜谱，镜像是做好的菜。

### 仓库（Registry）vs 本地
- **Registry（仓库）**：存镜像的云端服务，如 **Docker Hub**（公共默认）、私有仓库（阿里云/腾讯云/自建 Harbor）。
- `docker pull` 从仓库下镜像到本地；`docker push` 把本地镜像推上去。
- 镜像名形如 `nginx:1.25`（`仓库名:标签`，标签不写默认 `latest`）。

### 分层存储（Layered Storage）—— 效率的秘密
- 镜像由**多层只读层**叠加而成，每条 Dockerfile 指令产生一层。
- 多个镜像**共享相同层**（都基于 ubuntu:22.04 就共享那层），省磁盘和带宽。
- 容器在镜像之上加一层**可写层**（容器停止后该层仍存，删容器才消失）。

### 容器 vs 虚拟机（VM）
| 维度 | 容器 | 虚拟机 |
| --- | --- | --- |
| 隔离方式 | 共享宿主内核，进程级隔离 | 各自带完整操作系统 |
| 体积/启动 | MB 级、秒级启动 | GB 级、分钟级 |
| 资源占用 | 轻量 | 重 |

> 容器不是迷你虚拟机——它复用宿主机内核，隔离靠 Linux 的 namespace + cgroup（见 [[操作系统概览]]），所以又轻又快。

## 代码示例（命令）

```bash
# 1) 拉取公共镜像（从 Docker Hub）
docker pull nginx:1.25            # 仓库名:标签
docker pull python:3.12-slim

# 2) 查看本地镜像 / 容器
docker images                     # 本地镜像列表
docker ps                         # 运行中的容器
docker ps -a                      # 全部容器（含已停止）

# 3) 用镜像跑一个容器（实例化）
docker run -d --name web -p 8080:80 nginx:1.25
#   -d        后台运行
#   --name    给容器起名
#   -p 宿主:容器  端口映射（访问 localhost:8080 → 容器内 80）

# 4) 容器与镜像的关系
docker run -d --name web2 nginx:1.25   # 同一镜像再跑一个实例
# web 和 web2 互相隔离，但共享 nginx:1.25 镜像层

# 5) 分层结构查看
docker history nginx:1.25         # 看镜像每一层（每条指令）
```

## ⚠️ 易错点 / 面试要点

- **镜像 ≠ 容器**：高频面试题。镜像是只读模板，容器是运行实例。改了容器内的文件，**镜像不变**，只存在该容器的可写层。
- **删容器丢数据**：容器的可写层随容器删除而消失。要持久化必须用 volume/bind mount（见 [[数据卷与网络]]）。
- **`latest` 标签不可靠**：`nginx:latest` 会随时间指向不同版本，生产应**固定明确版本**（如 `nginx:1.25.3`），保证可复现。
- **容器不是虚拟机**：它共享内核，不能在 Linux 宿主上直接跑 Windows 容器（需特定方案）。隔离也比 VM 弱（同内核）。
- **分层决定构建效率**：Dockerfile 指令顺序影响缓存命中——把「不变层放前面、常变层放后面」能加速重复构建（见 [[镜像与 Dockerfile]]）。
- **公共镜像来源**：Docker Hub 是默认公共仓库，国内拉取可能慢，可配镜像加速器或用阿里云等。
