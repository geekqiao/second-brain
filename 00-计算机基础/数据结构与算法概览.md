# 数据结构与算法概览

> [!info] 广度概览 · 回到 [[🗺️ 知识地图 MOC]]

## 是什么

- **数据结构（Data Structure）**：组织和存储数据的方式，决定了存取、修改的效率。
- **算法（Algorithm）**：解决问题的有限步骤序列。
- 二者一体：**程序 = 数据结构 + 算法**（Niklaus Wirth）。选对结构，算法自然简单。

## 为什么重要

它回答软件最根本的工程问题——**在有限的时间和空间内把事做完**。同样的需求，`O(n²)` 和 `O(n log n)` 在数据量大时是天壤之别；这是面试和真实性能优化的核心。

## 核心要点

### 复杂度（Big-O）——先看这个再看代码
衡量输入规模 `n` 增长时代价如何增长。常见阶（从快到慢）：

| 阶 | 名称 | 直观感受（n=1e6） |
| --- | --- | --- |
| `O(1)` | 常数 | 瞬时 |
| `O(log n)` | 对数 | 极快 |
| `O(n)` | 线性 | 可接受 |
| `O(n log n)` | 线性对数 | 排序的标杆 |
| `O(n²)` | 平方 | 百万级会很慢 |
| `O(2ⁿ)` | 指数 | n=30 就爆 |

### 常见数据结构（一句话）
- **Array / List**：连续存储，随机访问 O(1)，插入删除慢。
- **Stack（栈）**：后进先出 LIFO，用于函数调用、撤销。
- **Queue（队列）**：先进先出 FIFO，用于任务调度、BFS。
- **Hash Table（哈希表）**：键值映射，平均 O(1) 查找；代价是哈希冲突。
- **Tree（树）**：层级结构，二叉搜索树 / 堆 / 红黑树；用于索引、优先级。
- **Graph（图）**：节点 + 边，有向/无向；用于网络、依赖、社交关系。

### 算法家族（知道有这些武器即可）
- **排序**：快排、归并（O(n log n)）；冒泡/选择（O(n²)，仅教学）。
- **查找**：二分查找（前提：有序）。
- **递归**：函数调用自身；注意栈深度。
- **分治（Divide & Conquer）**：拆分 → 解决 → 合并（如归并排序）。
- **动态规划（DP）**：重叠子问题 + 最优子结构，用记忆化避免重复计算。
- **贪心（Greedy）**：每步取局部最优，不一定全局最优但简单高效。

> [!tip] 学习心态
> 不要死记代码。先理解**每种结构擅长什么**，遇到问题先想「用什么结构存、用什么算法查」，复杂度自然就对了。
