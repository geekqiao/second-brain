# 计算机网络概览

> [!info] 广度概览 · 回到 [[🗺️ 知识地图 MOC]]

## 是什么

计算机网络是**让多台计算机互相通信、共享资源的系统**。它的全部巧妙在于一套**分层的协议**——每一层只解决一类问题，并为上一层服务。

## 为什么重要

只要你的程序联网（前端调 API、后端连数据库、微服务互调、上传文件），背后都是这些协议。理解网络，才能排查「为什么请求慢、为什么超时、为什么跨域报错」。

## 核心要点

### 两个分层模型（对照看）
| OSI 七层 | TCP/IP 四层 | 典型协议 | 解决的问题 |
| --- | --- | --- | --- |
| 应用层 | 应用层 | HTTP/HTTPS, DNS, SMTP | 业务怎么表达 |
| 表示层 | ↑ | TLS/SSL, JSON | 数据怎么编码加密 |
| 会话层 | ↑ | — | 会话管理 |
| 传输层 | 传输层 | **TCP / UDP** | 端到端可靠/不可靠传输 |
| 网络层 | 网际层 | **IP**, ICMP | 跨网络寻址路由 |
| 数据链路层 | 网络接口层 | Ethernet, ARP | 相邻节点帧传输 |
| 物理层 | ↑ | 电信号/光 | 比特怎么传输 |

> 实际工作记忆 **TCP/IP 四层** + **应用/传输/网络** 三层即可。

### TCP vs UDP
- **TCP**：面向连接、可靠、有序、有拥塞控制。用于网页、API、文件传输。
- **UDP**：无连接、不可靠、快、开销小。用于视频流、游戏、DNS 查询。

### HTTP / HTTPS 基础
- 请求-响应模型：方法（GET/POST/PUT/DELETE）+ 状态码（200/301/404/500）。
- HTTPS = HTTP + TLS，解决加密与身份认证。

### DNS：域名怎么变成 IP
`浏览器输入网址` → 查本地缓存 → 问 DNS 服务器（递归）→ 拿到 IP → 建立 TCP 连接 → 发 HTTP 请求。

### IP 与端口
- **IP 地址**定位「哪台主机」；**端口号**（0~65535）定位「主机上哪个进程」。
