# 数据库基础

> [!info] 广度概览 · 回到 [[🗺️ 知识地图 MOC]]

## 是什么

数据库（Database）是**有组织、可高效存取、可持久化的数据集合**。数据库管理系统（DBMS）则是管理它的软件。程序运行时数据在内存里（会丢），持久化就需要数据库。

## 为什么重要

- 几乎所有真实应用的核心都是数据：用户、订单、内容、日志。
- 选型（关系型 vs NoSQL）、建表、写查询、加索引、保事务——直接决定应用的正确性与性能。

## 核心要点

### 两大阵营
- **关系型（RDBMS）**：MySQL / PostgreSQL / SQLite。表 + 行 + 关系，用 **SQL** 查询，强结构、强一致。
- **NoSQL**：为特定场景而生——
  - 键值（Redis）：缓存、高速读写。
  - 文档（MongoDB）：灵活结构，存 JSON 式数据。
  - 列族（Cassandra）：海量数据、高写入。
  - 图（Neo4j）：关系密集（社交、推荐）。

### SQL 四大类（先记住动词）
| 类别 | 作用 | 关键字 |
| --- | --- | --- |
| DDL | 定义结构 | `CREATE / ALTER / DROP` |
| DML | 操作数据 | `INSERT / UPDATE / DELETE` |
| DQL | 查询数据 | `SELECT` |
| DCL | 控制权限 | `GRANT / REVOKE` |

### ACID 事务
事务（transaction）是把多个操作打包成「要么全成功、要么全回滚」的单元：
- **A**tomicity 原子性、**C**onsistency 一致性、**I**solation 隔离性、**D**urability 持久性。
- 典型场景：转账——扣款和加款必须同时成功或同时失败。

### 索引（index）为什么快
- 类似书的目录：用额外空间换查询速度，把全表扫描 O(n) 降到 O(log n)。
- 代价：写入变慢（要维护索引）、占额外空间。不是越多越好。

### Join（连接）
把多张表按某字段关联起来查询：内连接（交集）、左/右连接（保留一侧全部）。

> [!tip] 选型直觉
> 结构稳定、关系多、要强一致 → 关系型；结构多变、超高并发读写、可接受最终一致 → NoSQL。
